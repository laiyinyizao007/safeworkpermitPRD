## 常见问题与解决方案

### Q1: 如何避免文档过于技术化？
**问题**：产品文档写成了技术文档，缺少业务价值阐述。

**解决方案**：
- 保持"为什么-是什么-怎么做"的叙述结构
- 先讲业务价值，再讲技术实现
- 技术细节适度，聚焦于"做什么"而非"怎么做"
- 多用业务场景和用户故事

### Q2: 如何确保术语一致性？
**问题**：同一概念在不同章节使用不同术语，造成理解混乱。

**解决方案**：
- 在PROJECTWIKI.md中建立术语表
- 文档生成前先确定核心术语
- 使用codex检查术语一致性
- 必要时使用查找替换统一术语

### Q3: 如何平衡理想与现实？
**问题**：产品设计过于理想化，忽视实施约束。

**解决方案**：
- 采用分阶段策略（MVP → V2.0 → V3.0）
- 在PRD中明确约束条件
- 在Roadmap中标注风险和应对
- 提供备选方案和降级策略
