# 05 - 通用底座功能需求

## 5.1 基础信息管理

### 5.1.1 组织架构管理

**功能描述：**
支持多层级组织架构配置，包括公司、分厂、车间、班组等层级。

**核心功能：**
- 组织树形结构维护（增删改查）
- 组织层级关系配置（最多支持10级）
- 组织负责人设置
- 组织权限继承规则

**数据模型：**
```
Organization {
  id: string
  name: string
  code: string
  parentId: string
  level: number
  leaderId: string
  status: enum(active, inactive)
}
```

### 5.1.2 区域位置管理

**功能描述：**
管理企业内部作业区域，支持区域分级、坐标定位、风险等级标注。

**核心功能：**
- 区域信息维护（名称、编码、坐标、面积）
- 区域分类（生产区、储罐区、装卸区、办公区等）
- 风险等级标注（高危区、一般区、安全区）
- 区域平面图上传与标注
- 区域边界坐标配置（支持多边形）

**数据模型：**
```
Area {
  id: string
  name: string
  code: string
  type: enum
  riskLevel: enum(high, medium, low)
  boundary: GeoJSON
  floorPlan: string (图片URL)
}
```

### 5.1.3 设备设施管理

**功能描述：**
管理企业内部设备设施台账，支持设备分类、状态跟踪、维保记录。

**核心功能：**
- 设备台账维护（名称、型号、位置、状态）
- 设备分类（反应釜、储罐、泵、阀门等）
- 设备状态管理（运行、停机、检修、报废）
- 设备维保记录
- 设备关联作业历史

**数据模型：**
```
Equipment {
  id: string
  name: string
  code: string
  type: enum
  location: string (区域ID)
  status: enum
  lastMaintenanceDate: date
}
```

## 5.2 人员资质管理

### 5.2.1 人员信息管理

**功能描述：**
管理企业内部人员基础信息，支持人员分类、岗位配置、状态跟踪。

**核心功能：**
- 人员档案维护（姓名、工号、部门、岗位）
- 人员分类（作业人、监护人、审批人、管理员）
- 人员状态管理（在职、离职、休假、培训）
- 人员照片上传（用于人脸识别）
- 紧急联系人信息

**数据模型：**
```
User {
  id: string
  name: string
  employeeNo: string
  orgId: string
  position: string
  roles: array
  status: enum
  photo: string
  phone: string
  emergencyContact: object
}
```

### 5.2.2 资质证书管理

**功能描述：**
管理人员特种作业证书、培训证书，支持证书到期提醒、自动校验。

**核心功能：**
- 证书信息维护（证书类型、编号、有效期）
- 证书类型配置（电工证、焊工证、高处作业证等）
- 证书到期预警（提前30天、15天、7天）
- 证书扫描件上传
- 证书自动校验（作业申请时）

**数据模型：**
```
Certificate {
  id: string
  userId: string
  type: enum
  certNo: string
  issueDate: date
  expiryDate: date
  issuer: string
  attachment: string
  status: enum(valid, expired, revoked)
}
```

### 5.2.3 培训记录管理

**功能描述：**
记录人员安全培训历史，支持培训计划、考核成绩、证书发放。

**核心功能：**
- 培训计划制定（培训内容、时间、讲师）
- 培训签到记录
- 培训考核成绩
- 培训证书发放
- 培训统计分析

**数据模型：**
```
Training {
  id: string
  title: string
  type: enum
  startDate: date
  endDate: date
  participants: array
  instructor: string
  examScore: number
  certificateIssued: boolean
}
```

## 5.3 JSA风险库

### 5.3.1 风险库管理

**功能描述：**
建立标准化风险知识库，支持风险分类、等级评估、控制措施配置。

**核心功能：**
- 风险项维护（风险描述、类型、等级）
- 风险分类（火灾爆炸、中毒窒息、高处坠落等）
- 风险等级评估（LEC法：L×E×C）
- 控制措施配置（工程措施、管理措施、PPE）
- 风险库导入导出（Excel模板）

**数据模型：**
```
RiskItem {
  id: string
  name: string
  category: enum
  description: string
  likelihood: number (L)
  exposure: number (E)
  consequence: number (C)
  riskLevel: number (L×E×C)
  controlMeasures: array
}
```

### 5.3.2 JSA模板管理

**功能描述：**
为不同作业类型配置JSA分析模板，支持模板复用、快速填写。

**核心功能：**
- JSA模板配置（作业步骤、风险点、控制措施）
- 模板分类（按作业类型）
- 模板版本管理
- 模板引用统计
- 模板效果评估

**数据模型：**
```
JSATemplate {
  id: string
  workType: enum
  steps: array [
    {
      stepNo: number
      description: string
      risks: array
      controls: array
    }
  ]
  version: string
  usageCount: number
}
```

### 5.3.3 智能风险推荐

**功能描述：**
基于作业类型、区域、设备等信息，智能推荐相关风险点和控制措施。

**核心功能：**
- 基于规则的风险匹配
- 历史作业风险学习
- 风险推荐排序（按相关度）
- 推荐准确率统计
- 用户反馈收集

**推荐算法：**
```
推荐得分 = 作业类型匹配度 × 0.4
         + 区域风险匹配度 × 0.3
         + 历史频率 × 0.2
         + 季节因素 × 0.1
```

## 5.4 IoT接口层

### 5.4.1 设备接入管理

**功能描述：**
支持多种IoT设备接入，提供统一设备管理界面。

**核心功能：**
- 设备注册（设备ID、类型、协议）
- 设备认证（证书、密钥）
- 设备状态监控（在线、离线、故障）
- 设备配置下发
- 设备固件升级

**支持设备类型：**
- 气体检测仪（可燃气体、有毒气体、氧气）
- 视频监控（摄像头、NVR）
- 定位设备（UWB标签、蓝牙信标）
- 智能穿戴（安全帽、手环）
- 环境监测（温湿度、风速风向）

### 5.4.2 数据采集与处理

**功能描述：**
实时采集IoT设备数据，进行清洗、转换、存储。

**核心功能：**
- 数据采集（MQTT/HTTP订阅）
- 数据清洗（异常值过滤）
- 数据转换（单位换算、格式统一）
- 数据存储（InfluxDB时序库）
- 数据查询（按时间、设备、指标）

**数据流：**
```
IoT设备 → MQTT Broker → 数据清洗 → InfluxDB → API查询
```

### 5.4.3 告警规则配置

**功能描述：**
配置IoT设备告警规则，支持多级告警、自动通知。

**核心功能：**
- 告警规则配置（阈值、持续时间、告警级别）
- 告警通知配置（短信、邮件、APP推送）
- 告警升级机制（超时未处理自动升级）
- 告警屏蔽（维护期间）
- 告警统计分析

**告警级别：**
- 紧急（红色）：立即处理，自动停止作业
- 重要（橙色）：30分钟内处理
- 一般（黄色）：2小时内处理
- 提示（蓝色）：仅记录

## 5.5 定位引擎

### 5.5.1 人员定位

**功能描述：**
实时定位作业人员位置，支持轨迹回放、电子围栏。

**核心功能：**
- 实时定位（精度≤1米）
- 历史轨迹查询
- 电子围栏（进入/离开告警）
- 人员聚集检测
- 紧急呼救定位

**定位技术：**
- 室外：GPS + 北斗
- 室内：UWB + 蓝牙信标
- 融合定位：多技术融合，提高精度

### 5.5.2 设备定位

**功能描述：**
定位移动设备（如吊车、叉车）位置，支持运动轨迹分析。

**核心功能：**
- 设备实时位置
- 设备运动轨迹
- 设备作业范围限制
- 设备碰撞预警
- 设备利用率统计

### 5.5.3 空间分析

**功能描述：**
基于定位数据进行空间分析，支持SIMOPs检测。

**核心功能：**
- 空间距离计算
- 区域重叠检测
- 人员密度分析
- 危险源距离计算
- 疏散路径规划

## 5.6 SIMOPs冲突检测

详见：[docs/architecture/simops-algorithm.md](../docs/architecture/simops-algorithm.md)

### 5.6.1 空间冲突检测

**功能描述：**
检测多个作业的空间范围是否重叠或过近。

**检测规则：**
- 作业区域完全重叠 → 高风险冲突
- 作业区域部分重叠 → 中风险冲突
- 作业区域距离<安全距离 → 低风险冲突

**安全距离标准：**
- 动火作业：30米
- 受限空间作业：10米
- 高处作业：垂直5米、水平10米
- 吊装作业：吊装半径+10米

### 5.6.2 时间冲突检测

**功能描述：**
检测多个作业的时间段是否重叠。

**检测规则：**
- 时间完全重叠 → 高风险
- 时间部分重叠 → 中风险
- 时间相邻（间隔<1小时）→ 低风险

### 5.6.3 逻辑冲突检测

**功能描述：**
基于业务规则检测作业之间的逻辑冲突。

**冲突规则库：**
- 动火作业 + 受限空间作业（相邻区域）→ 可燃气体泄漏风险
- 高处作业 + 地面作业（垂直重叠）→ 坠物伤人风险
- 临时用电 + 动火作业（同一区域）→ 电气火花引燃风险
- 盲板抽堵 + 动火作业（同一管线）→ 介质泄漏风险

## 5.7 审批引擎

### 5.7.1 审批流程配置

**功能描述：**
灵活配置审批流程，支持多级审批、条件审批、会签/或签。

**核心功能：**
- 审批节点配置（审批人、审批角色）
- 审批条件配置（按风险等级、作业类型）
- 审批方式配置（会签、或签、依次审批）
- 审批时限配置（超时自动升级）
- 审批流程版本管理

**审批流程示例：**
```
一级作业（高风险）：
  班组长 → 车间主任 → 安全科 → 分管领导

二级作业（中风险）：
  班组长 → 车间主任 → 安全科

三级作业（低风险）：
  班组长 → 车间主任
```

### 5.7.2 电子签名

**功能描述：**
支持CA电子签名，确保审批法律效力。

**核心功能：**
- CA证书集成（对接第三方CA机构）
- 签名验证（RSA-2048 / SM2国密）
- 时间戳服务（RFC 3161标准）
- 签名可视化（显示签名图章）
- 签名证书链验证

### 5.7.3 审批记录追溯

**功能描述：**
完整记录审批过程，支持审批历史查询、责任追溯。

**核心功能：**
- 审批日志记录（时间、审批人、意见、IP）
- 审批历史查询
- 审批统计分析（通过率、平均时长）
- 审批责任追溯
- 审批数据导出

## 5.8 数据共享层

### 5.8.1 数据接口

**功能描述：**
提供标准RESTful API，支持第三方系统集成。

**核心接口：**
- 作业票CRUD接口
- 人员信息查询接口
- 设备状态查询接口
- 风险库查询接口
- 统计数据接口

**接口规范：**
- 协议：HTTPS
- 认证：OAuth 2.0 / API Key
- 数据格式：JSON
- 限流：1000次/分钟
- 文档：Swagger / OpenAPI 3.0

### 5.8.2 数据同步

**功能描述：**
与企业现有系统（ERP、MES、EAM）进行数据同步。

**同步方式：**
- 实时同步（Webhook推送）
- 定时同步（每小时/每天）
- 手动同步（按需触发）

**同步内容：**
- 人员信息（从HR系统）
- 设备台账（从EAM系统）
- 组织架构（从OA系统）
- 生产计划（从MES系统）

### 5.8.3 数据导出

**功能描述：**
支持数据批量导出，满足监管报送、数据分析需求。

**导出格式：**
- Excel（.xlsx）
- CSV
- PDF
- JSON

**导出内容：**
- 作业票台账
- 统计报表
- 审批记录
- 告警日志
