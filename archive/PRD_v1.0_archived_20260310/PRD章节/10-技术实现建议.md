## 10.2 编码规范

### 10.2.1 Java编码规范

**基于阿里巴巴Java开发手册：**

**命名规范：**
- 类名：大驼峰（PascalCase），如 `HotWorkTicket`
- 方法名：小驼峰（camelCase），如 `createTicket()`
- 常量：全大写+下划线，如 `MAX_RETRY_COUNT`
- 包名：全小写，如 `com.example.ptw.hotwork`

**注释规范：**


### 10.2.3 数据库规范

**表命名：**
- 小写+下划线：`hot_work_ticket`
- 统一前缀：`ptw_` (如 `ptw_hot_work_ticket`)

**字段命名：**
- 小写+下划线：`ticket_id`, `created_at`
- 布尔字段：`is_` 前缀，如 `is_deleted`
- 时间字段：`_at` 后缀，如 `created_at`, `updated_at`

**索引命名：**
- 主键：`pk_表名`
- 唯一索引：`uk_表名_字段名`
- 普通索引：`idx_表名_字段名`
- 外键：`fk_表名_字段名`

**示例：**
```sql
CREATE TABLE ptw_hot_work_ticket (
  id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
  ticket_no VARCHAR(32) NOT NULL COMMENT '作业票编号',
  work_type VARCHAR(20) NOT NULL COMMENT '作业类型',
  status VARCHAR(20) NOT NULL COMMENT '状态',
  created_by BIGINT NOT NULL COMMENT '创建人ID',
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  is_deleted TINYINT NOT NULL DEFAULT 0 COMMENT '是否删除',

  UNIQUE KEY uk_ticket_no (ticket_no),
  INDEX idx_status (status),
  INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='动火作业票';
```

### 10.3.2 统一响应封装

```java
@Data
public class Result<T> {
    private Integer code;
    private String message;
    private T data;
    private Long timestamp;

    public static <T> Result<T> success(T data) {
        Result<T> result = new Result<>();
        result.setCode(200);
        result.setMessage("success");
        result.setData(data);
        result.setTimestamp(System.currentTimeMillis());
        return result;
    }

    public static <T> Result<T> fail(Integer code, String message) {
        Result<T> result = new Result<>();
        result.setCode(code);
        result.setMessage(message);
        result.setTimestamp(System.currentTimeMillis());
        return result;
    }
}
```

## 10.4 数据库设计建议

### 10.4.1 分库分表策略

**分库：**
- 按企业ID分库（支持多租户）
- 每个企业独立数据库，数据隔离

**分表：**
- 作业票表按年份分表（如 `ptw_ticket_2026`, `ptw_ticket_2027`）
- IoT数据按月分表（如 `iot_data_202603`, `iot_data_202604`）

**路由规则：**
```java
@Component
public class DataSourceRouter {
    public String getDataSource(String enterpriseId) {
        return "ds_" + enterpriseId;
    }

    public String getTableName(String baseTable, LocalDate date) {
        return baseTable + "_" + date.getYear();
    }
}
```

### 10.4.2 索引优化

**常用查询索引：**
```sql
-- 作业票状态查询
CREATE INDEX idx_status_created_at ON ptw_ticket(status, created_at);

-- 作业人员查询
CREATE INDEX idx_worker_id ON ptw_ticket_worker(worker_id);

-- 审批流程查询
CREATE INDEX idx_approver_status ON ptw_approval(approver_id, status);

-- 时间范围查询
CREATE INDEX idx_work_time ON ptw_ticket(work_start_time, work_end_time);
```

**覆盖索引：**
```sql
-- 列表查询覆盖索引
CREATE INDEX idx_list_query ON ptw_ticket(
  status, created_at, ticket_no, work_type, location
);
```

### 10.4.3 数据归档策略

**归档规则：**
- 已完成作业票：6个月后归档
- 归档数据：迁移到历史库
- 保留在线数据：最近6个月

**归档脚本：**
```sql
-- 创建归档表
CREATE TABLE ptw_ticket_archive LIKE ptw_ticket;

-- 归档数据
INSERT INTO ptw_ticket_archive
SELECT * FROM ptw_ticket
WHERE status = 'completed'
  AND completed_at < DATE_SUB(NOW(), INTERVAL 6 MONTH);

-- 删除已归档数据
DELETE FROM ptw_ticket
WHERE status = 'completed'
  AND completed_at < DATE_SUB(NOW(), INTERVAL 6 MONTH);
```

## 10.5 缓存策略

### 10.5.1 缓存层次

**L1缓存（本地缓存）：**
- 使用Caffeine
- 缓存热点数据（用户信息、权限、配置）
- TTL：5分钟

**L2缓存（分布式缓存）：**
- 使用Redis
- 缓存业务数据（作业票、审批流程）
- TTL：30分钟

**示例：**
```java
@Service
public class TicketService {

    @Cacheable(value = "ticket", key = "#ticketId")
    public Ticket getTicket(String ticketId) {
        return ticketRepository.findById(ticketId);
    }

    @CacheEvict(value = "ticket", key = "#ticketId")
    public void updateTicket(String ticketId, Ticket ticket) {
        ticketRepository.save(ticket);
    }
}
```

### 10.5.2 缓存更新策略

**Cache Aside模式：**
1. 读取：先查缓存，未命中则查数据库，写入缓存
2. 更新：先更新数据库，再删除缓存

**防止缓存穿透：**
```java
public Ticket getTicket(String ticketId) {
    // 1. 查缓存
    Ticket ticket = redisTemplate.opsForValue().get("ticket:" + ticketId);
    if (ticket != null) {
        return ticket;
    }

    // 2. 查数据库
    ticket = ticketRepository.findById(ticketId);
    if (ticket == null) {
        // 缓存空对象，防止穿透
        redisTemplate.opsForValue().set("ticket:" + ticketId,
            new Ticket(), 5, TimeUnit.MINUTES);
        return null;
    }

    // 3. 写入缓存
    redisTemplate.opsForValue().set("ticket:" + ticketId,
        ticket, 30, TimeUnit.MINUTES);
    return ticket;
}
```

**防止缓存雪崩：**
- 设置随机过期时间
- 使用互斥锁
- 启用Redis持久化

**链路追踪（TraceId）：**
```java
@Component
public class TraceIdFilter implements Filter {
    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain) {
        String traceId = UUID.randomUUID().toString().replace("-", "");
        MDC.put("traceId", traceId);
        try {
            chain.doFilter(request, response);
        } finally {
            MDC.remove("traceId");
        }
    }
}
```
