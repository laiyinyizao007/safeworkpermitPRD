# 03 - 统一界面设计

> **设计理念**: 一套 UI，权限驱动差异。所有角色共享同一套界面，通过 `roleViewConfig` 动态控制字段可见性、可编辑性和操作按钮。
> **关联PRD**: [`09-用户体验设计.md`](../../产出/PRD章节/09-用户体验设计.md) / [`DOB NOW 集成设计`](../../docs/architecture/dob-now-integration.md)

---

## 1. 核心设计原则

| 原则 | 说明 |
|------|------|
| 一个页面，多种视图 | 作业票详情页只有一个，通过权限配置控制不同角色看到的内容 |
| 动作块模式 | 页面底部操作区根据角色 + 状态动态渲染可用操作（DOB NOW 理念） |
| 角色切换 = 权限刷新 | 切换角色后同一页面重新计算字段权限，不跳转到另一套页面 |
| 交互适配 ≠ 独立 UI | 作业人的大字大按钮、监护人的常驻叫停是交互层适配，不是独立页面 |
| 管理后台是功能模块 | 模板配置器、工作流设计器是管理员专属功能模块，不是"另一套 UI" |

## 4. 交互适配层

以下是同一 UI 框架下针对不同角色使用场景的交互适配，不是独立页面。

### 4.1 作业人适配 — 极简模式

当角色为"作业人"时，UI 框架自动应用：
- 正文字号 ≥ 16px，按钮高度 ≥ 48px
- 触摸区域 ≥ 44×44px（适配手套操作）
- 签名区域自动切换横屏全屏
- 信息精简：只显示与当前操作相关的内容

### 4.2 监护人适配 — 常驻面板模式

当角色为"监护人"且状态为 Executing 时：
- 🔴 紧急叫停按钮 `position: fixed` 底部，任何滚动位置可见
- 气体检测到期自动弹窗提醒 + 震动
- GPS 每 30 分钟自动打卡（无需手动操作）
- 低功耗设计：减少动画，降低刷新频率

### 4.3 安全负责人适配 — 效率模式

当角色为"安全负责人"且终端为 PC 时：
- 左右分栏布局（左侧详情 + 右侧审核操作）
- 异常项自动标红，合规项标绿
- 支持批量操作和快捷键
- 待审核列表支持"快速通过"（无异常时一键通过）

### 4.4 审批人适配 — 风险前置模式

当角色为"审批人"时：
- 风险摘要卡片置顶（风险等级 + 审核结论 + 自动校验结果）
- 详细信息默认折叠，聚焦决策所需核心数据
- 支持语音输入审批意见（🎤 长按录音转文字）
- 一级及以上强制地理围栏校验（100m 范围内）
