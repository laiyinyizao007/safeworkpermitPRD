> **定位**: 统一 UI 下各角色看到的差异速查表。程序员可直接将本文档映射到前端的 `v-if` / `v-show` / `:disabled` 逻辑。

## 1. 权限类型定义

| 图标 | 权限类型 | 前端映射 | 说明 |
| ---- | ---- | ---- | ---- |
| ✏️ | 读写 | `v-show + :disabled="false"` | 可见且可编辑 |
| 👁️ | 只读 | `v-show + :disabled="true"` | 可见但不可编辑 |
| 🚫 | 隐藏 | `v-if="false"` | 完全不渲染 |
| 📤 | 上传 | 上传组件启用 | 可上传文件/照片 |
| ✅ | 确认 | 勾选框启用 | 可勾选确认（不可修改内容） |
| ✍️ | 签名 | 签名组件启用 | 可进行电子签名 |

## 5. 角色切换交互说明

### 5.1 切换机制

- 首页右上角显示：`👤 姓名 [当前角色▼]`
- 点击角色标签弹出下拉菜单，列出该用户持有的所有角色
- 选择角色后，页面原地刷新（不跳转），重新按新角色计算权限
- Demo 中预置 6 个角色可自由切换，用于演示权限差异

### 5.2 切换后的变化

| 变化项 | 说明 |
| ---- | ---- |
| 待办列表 | 按新角色重新加载待办数据 |
| 详情页字段 | 按新角色重新计算可见/可编辑/隐藏 |
| 动作块 | 按新角色 + 当前状态重新渲染操作按钮 |
| 导航菜单 | 管理后台入口仅管理员可见 |
| 交互适配 | 自动应用新角色的交互适配（如作业人的大字模式） |

### 5.3 技术实现要点

```json
{
  "currentRole": "applicant",
  "availableRoles": ["applicant", "operator", "guardian", "reviewer", "approver", "admin"],
  "onRoleSwitch": "重新计算 roleViewConfig → 触发页面响应式更新"
}
```

## 6. 元数据权限配置格式（技术参考）

所有权限通过 JSON 元数据定义，渲染引擎在运行时根据 `role + state` 计算字段权限：

```json
{
  "field_key": "gas_detection_records",
  "permissions": {
    "applicant":  { "visible": true,  "editable": false },
    "operator":   { "visible": true,  "editable": false },
    "guardian":   { "visible": true,  "editable": true  },
    "reviewer":   { "visible": true,  "editable": false, "can_validate": true },
    "approver":   { "visible": true,  "editable": false },
    "admin":      { "visible": true,  "editable": true  }
  },
  "state_override": {
    "Draft":     { "guardian": { "visible": false } },
    "Verify":    { "guardian": { "editable": true } },
    "Executing": { "guardian": { "editable": true } },
    "Closed":    { "ALL": { "editable": false } }
  }
}
```

**渲染逻辑**: `最终权限 = permissions[role] ← merge ← state_override[state][role]`
