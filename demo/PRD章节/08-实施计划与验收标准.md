# 08 - 实施计划与验收标准

> **关联PRD**: [`11-项目实施计划.md`](../../产出/PRD章节/11-项目实施计划.md) / [`12-风险与应对.md`](../../产出/PRD章节/12-风险与应对.md)
> **核心理念**: 开发一套统一 UI + 一个权限引擎，不是开发 6 套界面。开发阶段按**页面/功能模块**组织，不按角色拆分。

---

## 1. 开发阶段划分

### Phase 1: 项目初始化（2天）

- 创建 Vue 3 + Vite + TypeScript 项目
- 安装依赖（Element Plus、Vant、Vue Router、Pinia、ECharts）
- 配置 TypeScript、ESLint、Prettier
- 配置移动端适配（Viewport + Flexible）
- 创建项目目录结构（按功能模块组织，非按角色）
- 配置统一路由（`/`、`/permit/new`、`/permit/:id`、`/admin/*`）

### Phase 2: Mock 数据与类型定义（2天）

- 定义 TypeScript 类型（User、WorkPermit、FormMetadata、Permission）
- 编写 6 个用户 Mock 数据
- 编写 8 张不同状态的作业票 Mock 数据
- 编写动火作业表单元数据配置（含角色权限）
- 编写气体检测、审计日志、统计数据 Mock
- 创建 Mock API 服务层

### Phase 3: 通用组件开发（3天）

- 动态表单渲染器（FormRenderer）— 含权限控制
- 电子签名组件（Signature）— Canvas API
- 照片上传组件（PhotoUpload）— 自动压缩 + 水印
- 气体检测录入组件（GasDetection）— 数字键盘 + 超标报警
- 紧急叫停按钮（EmergencyStop）— 常驻底部
- 状态标签组件（StatusTag）
- 审批时间轴组件（Timeline）
- 地理围栏模拟组件（GeoFence）
- 角色切换组件（RoleSwitcher）

### Phase 4: 统一页面开发 — 核心业务页面（5天）

> 所有页面共享同一套组件，通过权限引擎控制不同角色看到的字段和操作按钮。

**Day 1: 首页（待办中心）**
- 统一待办列表（所有角色共享，按角色过滤待办数据）
- 状态筛选 + 搜索
- 角色切换组件（RoleSwitcher）集成
- 新建作业票入口（仅负责人/管理员可见）

**Day 2: 新建作业票（分步表单）**
- 4 步分步表单（基础信息 → 人员 → 安全措施 → 确认提交）
- FormRenderer 集成（元数据驱动字段渲染）
- 表单校验 + 草稿保存

**Day 3-4: 作业票详情页（核心页面，权限驱动）**
- 统一详情页框架（所有角色共享同一个 Detail.vue）
- 权限引擎集成：根据 `role + status` 计算字段可见/可编辑/隐藏
- 动作块（ActionBlock）：根据角色×状态动态渲染操作按钮
  - 负责人动作：编辑、提交、撤回、验收签字
  - 作业人动作：确认措施、签到、签退
  - 监护人动作：核查通过/退回、气体检测录入、紧急叫停、验收签字
  - 安全负责人动作：审核通过/退回
  - 审批人动作：现场审批、远程审批、紧急审批
- 审批时间轴（Timeline）
- 监护面板（条件渲染，仅监护人在 Executing 状态可见）

**Day 5: 交互适配层**
- 作业人极简模式（大字大按钮 CSS 适配）
- 监护人常驻面板（紧急叫停 fixed 定位 + 气体检测定时弹窗）
- 审批人风险前置（风险摘要卡片置顶 + 详情折叠）
- 签名横屏全屏模式

### Phase 5: 管理后台 + PC 端适配（4天）

> 管理后台是唯一的角色专属功能模块（仅管理员可见），PC 端适配是布局层面的响应式处理。

**Day 1: 安全负责人 PC 端适配**
- 审核工作台（待审核列表 + 自动校验报告）— 基于统一详情页的 PC 宽屏布局
- 左右分栏布局（列表 + 详情）
- 数据看板（ECharts）

**Day 2: 审批人 PC 端适配 + 统计看板**
- 审批统计看板
- 手机端快速审核交互（左滑退回 / 右滑通过）

**Day 3-4: 系统管理员后台**
- 系统概览仪表盘
- 模板配置器（三栏拖拽布局）
- 工作流状态机设计器
- 权限矩阵编辑器
- 数据分析看板 + 审计日志

### Phase 6: 权限引擎与状态流转（2天）

- 权限引擎完整实现（`role × status → fieldPermission` 计算逻辑）
- 完整流程测试（申请 → 审核 → 审批 → 核查 → 执行 → 验收）
- 状态流转逻辑（Pinia 状态管理）
- 角色切换后同一页面权限刷新（不跳转，重新计算 roleViewConfig）
- 通知消息模拟（Toast + 消息列表）
- 紧急叫停 → 紧急中断状态流转
- 退回 → 修改 → 重新提交流转

### Phase 7: UI/UX 优化（2天）

- 移动端适配优化
- PC 端响应式布局
- 交互动画（页面切换、状态变化）
- 加载状态、空状态
- 错误提示、表单校验反馈
- 角色差异化交互适配微调（参见 [07-UI设计规范.md](./07-UI设计规范.md) 第4节）

### Phase 8: Demo 演示准备（1天）

- 准备演示脚本（6 角色完整流程）
- 准备演示数据（确保各状态覆盖）
- 部署到演示环境
- 演示彩排

**总计：约 21 个工作日（4 周）**

## 2. 验收标准

### 2.1 功能验收

- [ ] 6 个角色可通过顶部下拉菜单正常切换
- [ ] 作业负责人：完整的创建 → 提交 → 查看进度 → 验收流程
- [ ] 作业人：签到确认 → 执行 → 签退完整流程
- [ ] 监护人：现场核查 → 实时监护 → 气体检测 → 紧急叫停 → 验收
- [ ] 安全负责人：PC 端审核工作台 + 手机快速审核
- [ ] 审批人：现场审批 + 远程审批 + 紧急审批
- [ ] 系统管理员：模板配置器 + 工作流设计器 + 权限矩阵 + 数据看板 + 审计日志
- [ ] 状态流转逻辑正确（Draft → Submitted → Verify → Executing → Closed）
- [ ] 退回流程正确（退回 → 修改 → 重新提交）
- [ ] 紧急叫停流程正确（Executing → Emergency → Closed/Executing）
- [ ] 动态表单渲染正确（元数据驱动）
- [ ] 角色×状态权限控制正确（不同角色看到不同字段和操作）

### 2.2 体验验收

- [ ] 移动端适配良好（375px ~ 414px）
- [ ] PC 端布局合理（1024px+）
- [ ] 交互流畅，无卡顿
- [ ] 加载状态清晰
- [ ] 错误提示友好
- [ ] 视觉风格统一
- [ ] 作业人界面大字大按钮
- [ ] 监护人紧急叫停按钮常驻可见
- [ ] 安全负责人异常项高亮

### 2.3 演示验收

- [ ] 可完整演示 6 个角色的核心流程
- [ ] 演示数据准备充分（覆盖各状态）
- [ ] 演示脚本清晰（6 角色协作场景）
- [ ] 角色切换流畅自然
- [ ] 可在 15 分钟内完成一次完整演示

## 3. 风险与应对

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|
| 权限引擎复杂度高（6角色×6状态组合） | 开发周期延长 | 高 | 统一 UI 架构减少页面数量（~10页）；权限矩阵驱动，避免重复开发 |
| 动态表单渲染器复杂度高 | 开发难度大 | 中 | 先实现基础组件，复杂组件单独封装；使用 Element Plus 现成组件 |
| PC + 移动端双端适配 | 工作量翻倍 | 中 | 安全负责人/管理员仅 PC 端；作业人/监护人仅移动端；减少双端页面 |
| 模板配置器拖拽交互复杂 | 开发难度大 | 中 | 使用 vuedraggable 库；Demo 阶段仅展示交互，不实现真实保存 |
| 工作流设计器可视化复杂 | 开发难度大 | 中 | 使用 vue-flow 库；预置固定流程图，支持简单编辑 |
| 演示数据不充分 | 演示效果差 | 低 | 提前准备 8 张不同状态的作业票；每个角色至少 2 个演示场景 |

## 4. 演示脚本大纲

### 场景：一张动火作业票的完整生命周期

**开场（1分钟）**：
- 切换到系统管理员（陈工），展示系统概览仪表盘
- 简要展示模板配置器和工作流设计器

**第一幕：申请（3分钟）**：
- 切换到作业负责人（张三）
- 新建动火作业票，4 步填写
- 提交审批

**第二幕：审核（2分钟）**：
- 切换到安全负责人（赵六）
- PC 端审核工作台，查看自动校验报告
- 发现灭火器不足，退回修改
- 张三修改后重新提交，赵六审核通过

**第三幕：审批（2分钟）**：
- 切换到审批人（孙七）
- 查看待审批列表，进入现场审批
- 地理围栏校验，逐项确认，签名审批通过

**第四幕：核查与执行（3分钟）**：
- 切换到监护人（王五）
- 现场核查（安全措施逐项确认 + 气体检测）
- 核查通过，进入监护面板
- 切换到作业人（李四），签到确认
- 回到监护人，展示实时监护面板 + 气体检测录入

**第五幕：紧急叫停（1分钟）**：
- 监护人发现异常，点击紧急叫停
- 展示叫停确认流程和通知机制

**第六幕：验收关闭（1分钟）**：
- 作业人签退
- 监护人验收签字
- 作业负责人确认验收，关闭作业票

**收尾（1分钟）**：
- 切换到安全负责人，展示数据看板
- 切换到系统管理员，展示审计日志

**总时长：约 14 分钟**
