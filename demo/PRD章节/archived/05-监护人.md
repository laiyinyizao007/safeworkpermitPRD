# 05 - 监护人

> 角色详细分析参见：[03-监护人（角色视角）](../../分析内容/八大作业人员与工作流程/角色视角/03-监护人.md)

---

## 1. 角色画像

### 1.1 角色定位

全程在场的安全守门人 —— 从作业开始到结束，监护人必须始终在场，是现场安全的最后一道防线。

### 1.2 典型用户

- 持监护人证的安全员
- 班组安全骨干

### 1.3 主要终端

手机（常驻现场，不离手）

### 1.4 职责清单

| # | 职责 |
|---|------|
| 1 | 全程在场不脱岗 |
| 2 | 检查作业票与实际现场相符 |
| 3 | 录入气体检测数据 |
| 4 | 实时监控作业人员状态 |
| 5 | 发现异常立即叫停 |
| 6 | 作业完成参与验收 |

### 1.5 使用场景

| 场景 | 频率 | 终端 | 耗时 |
|------|------|------|------|
| 现场核查（Verify） | 每票 1 次 | 手机 | 10-20 分钟 |
| 气体检测录入 | 每 2 小时 1 次 | 手机 | 2-3 分钟 |
| 实时监护常驻 | 全程 | 手机 | 数小时 |
| 紧急叫停 | 极少 | 手机 | 即时 |
| 验收签字 | 每票 1 次 | 手机 | 3-5 分钟 |

### 1.6 痛点与设计对策

| 痛点 | 设计对策 |
|------|----------|
| 长时间现场手机耗电 | 轻量低功耗界面，减少动画和后台刷新 |
| 快速录入检测数据 | 数字键盘优先，IoT 蓝牙自动采集 |
| 紧急叫停需即时响应 | 叫停按钮常驻底部，无需翻页 |
| 被查岗需证明在场 | GPS 轨迹自动记录，30 分钟打卡 |

### 1.7 设计原则

- 常驻监护面板，信息一屏可见
- 一键叫停永远可见（红色大按钮 fixed 底部）
- 自动记录 GPS / 在岗时间，无需手动操作
- 定时提醒气体检测（弹窗 + 震动）

## 3. 完整用户流程

```mermaid
flowchart TD
    A[接收监护任务] --> B[前往作业现场]
    B --> C{GPS 定位校验}
    C -->|不在 50m 内| C1[提示：请到达现场]
    C1 --> C
    C -->|在 50m 内| D[现场核查]

    D --> D1[安全措施逐项核查 + 拍照]
    D1 --> D2[气体检测录入]
    D2 --> D3{检测数据是否合格?}
    D3 -->|超标| D4[自动报警，阻断通过]
    D4 --> D2
    D3 -->|合格| D5[电子签名]
    D5 --> D6{核查结果}
    D6 -->|不通过| D7[退回作业票，通知负责人]
    D6 -->|通过| E[开始监护]

    E --> F[实时监护面板]
    F --> G{监护循环}

    G --> G1[GPS 自动打卡<br/>每 30 分钟]
    G1 --> G

    G --> G2{气体检测到期?}
    G2 -->|是| G3[弹窗提醒录入]
    G3 --> G4{数据是否合格?}
    G4 -->|超标| G5[自动触发告警] --> H
    G4 -->|合格| G
    G2 -->|否| G

    G --> H{发现异常?}
    H -->|是| H1[紧急叫停]
    H1 --> H2[选择叫停原因 + 拍照]
    H2 --> H3[系统自动通知所有人]
    H3 --> H4[作业票锁定为紧急中断]
    H4 --> H5[等待安全科处理]

    H -->|否| I{作业是否完成?}
    I -->|否| G
    I -->|是| J[验收核查]

    J --> J1[验收清单逐项确认]
    J1 --> J2[最终气体检测]
    J2 --> J3[电子签名]
    J3 --> K[关闭作业票]
    K --> L[监护任务完成]
```

## 4. 通知与消息

| 通知事件 | 通知方式 | 优先级 |
|----------|----------|--------|
| 被指定为监护人 | 推送 + 短信 | 高 |
| 作业票审批通过，可开始核查 | 推送 | 中 |
| 气体检测到期提醒 | 弹窗 + 震动 | 紧急 |
| 作业即将到期（剩余 30 分钟） | 推送 + 震动 | 高 |
| GPS 偏离告警（离开 50m 范围） | 弹窗 + 震动 | 紧急 |
