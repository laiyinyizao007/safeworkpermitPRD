# 10 - RoleSwitcher 角色切换

> **组件路径**: `components/RoleSwitcher/` | **使用页面**: 全局（顶部导航） | **技术要点**: 下拉菜单, 6角色快速切换
> **关联**: [04-角色权限视图差异.md](../04-角色权限视图差异.md) §5

---

## 1. 组件定位

Demo 专用的角色快速切换组件。右上角下拉菜单，6 个角色可自由切换，无需重新登录。切换后页面原地刷新权限。

## 2. Props / Events 接口

```typescript
interface RoleSwitcherProps {
  currentUser: User
  users: User[]           // 可切换的用户列表（6人）
}

interface RoleSwitcherEvents {
  'switch': (user: User) => void
}
```

## 3. 视觉规格

- 触发器：`👤 姓名 [当前角色▼]`
- 下拉菜单：6 个选项，每项显示头像 + 姓名 + 角色名
- 当前角色高亮标记
- 切换动画：0.3s 淡入淡出

## 4. 交互行为

1. 点击角色标签弹出下拉菜单
2. 选择角色后：
   - 更新 Pinia `userStore` 中的当前用户
   - 页面原地刷新（不跳转），重新按新角色计算权限
   - Toast 提示"已切换为 XX（角色名）"

### 切换后的变化

| 变化项 | 说明 |
|--------|------|
| 待办列表 | 按新角色重新加载待办数据 |
| 详情页字段 | 按新角色重新计算可见/可编辑/隐藏 |
| 动作块 | 按新角色 + 当前状态重新渲染操作按钮 |
| 导航菜单 | 管理后台入口仅管理员可见 |
| 交互适配 | 自动应用新角色的交互适配（如作业人的大字模式） |

## 5. 使用场景

全局组件，所有页面顶部导航栏右侧。

## 6. 与其他组件的关系

- 切换角色后触发 FormRenderer 重新计算字段权限
- 切换角色后触发 ActionBlock 重新渲染操作按钮

## 7. Demo 简化说明

- 这是 Demo 专用组件，正式产品中不存在（正式产品通过登录确定角色）
- 6 个预置用户可自由切换
- 切换不需要密码验证
