# 01 - 首页 - 待办中心

> **路由**: `/` | **可见角色**: 所有角色 | **关联组件**: StatusTag, ActionBlock, RoleSwitcher
> **关联PRD**: [03-统一界面设计.md](../03-统一界面设计.md) §3
> **AI 集成**: Auditor Agent（待办卡片合规分数徽章）

---

## 1. 页面定位

所有角色登录后的统一入口。同一个页面框架，内容根据当前角色动态填充待办数据。

## 2. 功能模块

| 模块 | 说明 |
|------|------|
| 顶部导航栏 | 系统标题 + 角色切换下拉 + 消息通知 |
| 待办统计卡片 | 按角色显示不同维度的统计数据 |
| 待办事项列表 | 按角色过滤的作业票列表，每张卡片含动作块 |
| 新建入口 | [+ 新建作业票] 按钮，仅负责人/管理员可见 |

## 4. 角色差异（同一页面不同角色看到什么）

| 角色 | 待办统计内容 | 待办列表内容 | 动作块 | 特殊入口 |
|------|------------|------------|--------|---------|
| 作业负责人 | 我发起的票据统计 | 我发起的作业票 | 查看进度/催办/撤回 | [+ 新建作业票] |
| 作业人 | 分配给我的作业 | 待签到/进行中的作业 | 去签到/签退 | — |
| 监护人 | 我的监护任务 | 待核查/监护中的作业 | 开始核查/进入监护 | — |
| 安全负责人 | 待审核统计 | 待审核的作业票 | 审核/快速通过/快速退回 | — |
| 审批人 | 待审批统计 | 待审批的作业票 | 现场审批/远程审批 | — |
| 系统管理员 | 系统运行指标 | 系统告警/证书预警 | 进入管理后台 | [管理后台] |

## 5. 状态驱动 UI 变化

首页不直接受作业票状态驱动，但待办列表的内容和排序受状态影响：

| 优先级 | 条件 | 显示效果 |
|--------|------|---------|
| 最高 | Emergency 状态的票 | 🔴 红色高亮置顶 |
| 高 | 被退回需修改的票 | 🟠 橙色标记 |
| 中 | 待当前角色操作的票 | 🟡 黄色标记 |
| 低 | 进行中但不需当前角色操作 | 默认样式 |
| 提示 | 存在未满足的前置依赖 | ⚠️ 依赖警告（显示在状态行下方） |
| 提示 | 存在 SIMOPS 冲突 | ⚠️ 冲突警告（显示在状态行下方） |

## 6. 页面导航

| 操作 | 目标 |
|------|------|
| 点击待办卡片 | → 作业票详情页 `/permit/:id` |
| 点击 [+ 新建作业票] | → 新建作业票 `/permit/new` |
| 点击 [管理后台] | → 管理后台 `/admin` |
| 点击消息通知图标 | → Toast 消息列表（Demo 简化） |

## 7. Mock 数据接口

```typescript
// 获取待办统计
GET /api/home/statistics?role=:role
Response: { pending: number, inProgress: number, todayCompleted: number }

// 获取待办列表
GET /api/home/todos?role=:role
Response: WorkPermit[]  // 每个 WorkPermit 扩展字段:
// - ai_audit_score?: number (0-100) - AI 合规分数徽章
// - dependency_warning?: string - 依赖警告文本（例如: "等待 BP-001 完成"）
// - conflict_warning?: string - 冲突警告文本（例如: "与 CS-003 冲突，距 12m"）
// ai_audit_score 用于待办卡片右上角 AI 合规分数徽章
// Mock: 已提交/已审批的票据返回 70-98 随机分数，草稿状态不返回
// dependency_warning/conflict_warning: 根据 DependencyEngine 分析结果返回
```

## 8. 交互适配

| 角色 | 适配 |
