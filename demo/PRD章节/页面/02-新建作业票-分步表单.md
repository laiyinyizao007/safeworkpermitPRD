# 02 - 新建作业票 - 分步表单

> **路由**: `/permit/new` | **可见角色**: 作业负责人、系统管理员 | **关联组件**: FormRenderer, PhotoUpload, Signature, GeoFence, AIReviewSidebar, SafetyChecklist
> **关联PRD**: [03-统一界面设计.md](../03-统一界面设计.md) §4
> **AI 集成**: [14-AIReviewSidebar.md](../组件/14-AIReviewSidebar-AI审查侧边栏.md) / KG Reasoning Agent / Auditor Agent / SIMOPs Agent

---

## 1. 页面定位

作业票创建入口，采用分步表单引导用户完成信息填写。表单字段由元数据驱动，所有角色共享同一套表单引擎。仅作业负责人和管理员可见入口。

## 2. 功能模块

| 步骤 | 模块 | 核心字段 |
|------|------|---------|
| Step 1 | 选择作业类型 | 8大作业类型卡片选择 + 历史模板复制 |
| Step 2 | 基础信息 | 作业区域、地点、等级、方式、时间、内容、现场照片 |
| Step 3 | 人员信息 | 动火人员(持证校验)、监护人、审核/审批人(自动关联) |
| Step 4 | 安全措施 + 提交 | 安全措施清单、JSA风险分析、应急预案、🤖 AI推荐安全措施/风险项、AI审查、提交审批 |

#### 3.4.3 提交前验证流程（AI审查 + 依赖验证）

```text
触发方式 A: 手动点击 [AI审查 🤖] 按钮
  → 打开 AIReviewSidebar（PC右侧/移动端底部抽屉）
  → 调用 POST /api/ai/review（聚合 3 个智能体结果）
  → 展示审查项列表

触发方式 B: 点击 [提交审批 →] 按钮
  → Step 1: 自动触发 AI 审查
    → 有 CRITICAL 或 HIGH 问题？
      → 是: 阻断提交 + 自动打开 AIReviewSidebar + Toast 提示"请先处理严重问题"
      → 否: 继续 Step 2

  → Step 2: 🔗 依赖关系验证（DependencyEngine）
    → 调用 POST /api/tasks/{taskId}/permits/validate
    → 检查：
      ✓ 前置依赖是否满足（如：盲板抽堵是否已完成）
      ✓ SIMOPS冲突检测（与其他活动作业表的空间/时间冲突）
      ✓ 条件性依赖评估（气体检测、方案编制、监护人配置）
    → 有错误（Error）？
      → 是: 阻断提交 + 弹出依赖验证对话框 + Toast 提示"依赖关系未满足"
      → 否: 有警告（Warning）？
        → 是: 弹出确认对话框"检测到SIMOPS冲突警告，是否继续？"
          → 用户选择"继续" → Step 3
          → 用户选择"取消" → 返回编辑
        → 否: 继续 Step 3

  → Step 3: 正常提交流程
    → 提交成功 → 跳转到详情页
```

#### 3.4.4 KG 推荐安全措施交互

| 操作 | 行为 |

## 4. 角色差异

本页面仅作业负责人和管理员可访问，角色差异较小：

| 角色 | 差异 |
|------|------|
| 作业负责人 | 标准流程，审核/审批人自动关联 |
| 系统管理员 | 可编辑所有字段，可指定任意审核/审批人 |

## 5. 状态驱动 UI 变化

| 场景 | UI 变化 |
|------|---------|
| 新建 | 空白表单，所有字段可编辑 |
| 从草稿恢复 | 预填已保存数据，继续编辑 |
| 从退回修改进入 | 预填原数据，退回标注字段高亮，自动聚焦到需修改字段 |
| 从历史复制 | 预填历史数据，清空签名和时间 |

## 6. 页面导航

| 来源 | 操作 | 目标 |
|------|------|------|
| 首页 | 点击 [+ 新建作业票] | → 本页面 Step 1 |
| 详情页(Rejected) | 点击 [修改并重新提交] | → 本页面(预填+高亮) |
| 详情页(Closed) | 点击 [复制新建] | → 本页面(预填) |
| 本页面 | 提交成功 | → 作业票详情页 |
| 本页面 | 保存草稿 | → 首页 |
