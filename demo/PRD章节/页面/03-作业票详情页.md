## 1. 页面定位

整个系统最核心的页面。所有角色打开同一张作业票，看到的是同一个页面，区别在于：
- 哪些字段可见 / 隐藏
- 哪些字段可编辑 / 只读
- 底部动作块显示哪些操作按钮
- 是否显示状态特定的附加面板

**设计理念**: 一个页面，多种视图。权限差异通过 `roleViewConfig` 运行时计算，不是独立页面。

## 2. 功能模块

| 模块 | 说明 | 关联组件 |
|------|------|---------|
| 顶部状态栏 | 票号 + 状态标签 + 角色切换 | StatusTag, RoleSwitcher |
| 状态进度条 | 可视化流转进度 | Timeline |
| 基础信息卡片 | 作业区域/等级/方式/时间/内容 | FormRenderer |
| 人员信息卡片 | 动火人/监护人/审核审批人 | FormRenderer |
| 安全措施卡片 | 安全措施清单 + JSA 风险分析 | FormRenderer, SafetyChecklist |
| 气体检测卡片 | 检测记录（仅核查/执行阶段可见） | GasDetection |
| 现场照片卡片 | 照片展示/上传 | PhotoUpload |
| 审核/审批意见卡片 | 按状态逐步展示 | FormRenderer |
| 状态附加面板 | 退回原因/监护面板/审核面板等 | MonitorPanel, RiskSummary |
| 🤖 AI 合规审计面板 | Submitted 状态展示 AI 审计结果（替换静态校验报告） | AuditResultPanel |
| 🤖 多专家共识面板 | Approved 状态嵌入 RiskSummary，展示三专家评估 | ExpertConsensusPanel |
| 🤖 时空验证徽章 | 审批人现场审批前的位置校验 | SpatialVerifyBadge |
| 🤖 作业关联图谱 | 可折叠卡片，展示 KG 依赖关系 | — (内联渲染) |
| 🤖 数据推送状态 | Closed 状态展示数据推送结果 | — (内联渲染) |
| 底部动作块 | 按角色+状态动态渲染操作按钮 | ActionBlock |

## 4. 角色差异（同一页面不同角色看到什么）

### 4.1 动作块渲染规则

同一张票，不同角色在不同状态下看到不同的动作块：

| 状态 | 作业负责人 | 作业人 | 监护人 | 安全负责人 | 审批人 |
|------|----------|--------|--------|----------|--------|
| Draft | [编辑] [提交] [删除] | — | — | — | — |
| Submitted | [撤回] [催办] | — | — | [审核通过] [退回] | — |
| Rejected | [修改并重新提交] | — | — | — | — |
| Approved | (等待) | — | — | — | [现场审批] [远程审批] |
| Verify | [补充材料] | [确认措施] | [核查通过] [退回] + 🔴叫停 | — | — |
| Executing | (查看) | [签退] [报告异常] | [录入检测] [拍照] + 🔴叫停 | — | — |
| PendingAcceptance | [验收签字] | — | [验收签字] | — | — |
| Closed | [导出PDF] [复制新建] | — | — | [统计] | — |

### 4.2 字段权限渲染示例

以"安全措施"区块为例，同一个区块在不同角色下的渲染差异：

| 角色 | 安全措施清单 | 灭火器数量 | 气体检测 |
|------|------------|----------|---------|
| 作业负责人(Draft) | ✏️ 可勾选编辑 | ✏️ 可输入 | 🚫 隐藏 |
| 作业人(Verify) | 👁️ 只读 + ✅确认按钮 | 👁️ 只读 | 🚫 隐藏 |
| 监护人(Verify) | ✅ 逐项核查+拍照 | 👁️ 只读 | ✏️ 可录入 |
| 安全负责人(Submitted) | 👁️ 只读 + 🔴异常标注 | 👁️ 只读 + 自动校验 | 🚫 隐藏 |
| 审批人(Approved) | 👁️ 只读(折叠) | 👁️ 只读 | 🚫 隐藏 |

> 完整的字段级权限矩阵见 [04-角色权限视图差异.md](../04-角色权限视图差异.md)

### 4.3 交互适配差异

| 角色 | 适配策略 | 具体表现 |
|------|---------|---------|
| 作业人 | 极简模式 | 正文 ≥16px，按钮 ≥48px，触摸区 ≥44×44px，信息精简 |
| 监护人 | 常驻面板模式 | 🔴叫停按钮 fixed 底部，气体检测到期弹窗，GPS 自动打卡 |
| 安全负责人 | 效率模式(PC) | 左右分栏，异常标红，批量操作，快捷键 |
| 审批人 | 风险前置模式 | 风险摘要卡片置顶，详情默认折叠，语音输入，地理围栏校验 |

## 5. 状态驱动 UI 变化

| 状态 | 可写字段 | 只读字段 | 约束卡点 | 附加面板 |
|------|---------|---------|---------|---------|
| Draft | 基础信息、人员信息、安全措施 | — | — | — |

## 7. Mock 数据接口

```typescript
// 获取作业票详情（扩展 AI 字段）
GET /api/permits/:id
Response: WorkPermit & {
  ai_audit?: {                          // Submitted 状态返回
    compliance_score: number,
    check_items: AuditCheckItem[],
    suggestions: string[],
    audit_time: string
  },
  ai_expert_consensus?: {               // Approved 状态返回
    overall_risk: string,
    consensus_type: 'unanimous' | 'majority' | 'no_consensus',
    consensus_score: number,
    expert_judgments: ExpertJudgment[],
    final_recommendation: string,
    dissenting_opinions?: string[]
  }
}

// 更新作业票
PUT /api/permits/:id
Body: Partial<WorkPermit>

// 提交审批操作
POST /api/permits/:id/action
Body: { action: 'submit' | 'approve' | 'reject' | 'verify' | ... , comment?: string }

// 获取字段权限
GET /api/permits/:id/permissions?role=:role
Response: Record<string, FieldPermission>

// 🤖 时空验证（审批人现场审批前）
POST /api/spatial-temporal/verify-approval
```
