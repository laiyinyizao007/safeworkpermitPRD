> **路由**: `/permit/:id/sign` | **可见角色**: 需签名的角色（负责人、作业人、监护人、审批人） | **关联组件**: Signature, SpatialVerifyBadge
> **关联PRD**: [03-统一界面设计.md](../03-统一界面设计.md) §2
> **AI 集成**: [17-SpatialVerifyBadge.md](../组件/17-SpatialVerifyBadge-时空验证徽章.md) / Spatial-Temporal Agent

## 1. 页面定位

独立的全屏签名页面。从作业票详情页的签名区域跳转进入，签名完成后自动返回详情页。移动端强制横屏，最大化签名区域，适配手套操作场景。

## 2. 功能模块

| 模块 | 说明 |
|------|------|
| 签名画布 | Canvas 全屏绘制区域，支持鼠标/触摸 |
| 签名人信息 | 显示当前签名人姓名、角色、时间 |
| 操作按钮 | [清除重签] [确认签名] |
| 签名预览 | 确认前预览签名效果 |

## 4. 角色差异

| 角色 | 签名场景 | 特殊要求 |
|------|---------|---------|
| 作业负责人 | 提交时签名、验收时签名 | 标准签名 |
| 作业人 | 签到确认时签名 | 横屏全屏，大触摸区（适配手套） |
| 监护人 | 核查通过时签名、验收时签名 | 标准签名 |
| 审批人 | 审批通过时签名 | 支持 PC 端鼠标签名，🤖 签名前需通过时空验证 |

- 验证通过（✅）：签名区域正常可用

- 未验证（📍）：点击徽章触发验证

## 5. 状态驱动 UI 变化

签名页本身不受作业票状态驱动，但签名的触发时机与状态相关：

| 状态 | 触发签名的角色 | 签名用途 |
|------|-------------|---------|
| Draft → Submitted | 作业负责人 | 提交确认签名 |
| Approved → Verify | 审批人 | 审批确认签名 |
| Verify → Executing | 监护人、作业人 | 核查/签到确认签名 |
| PendingAcceptance → Closed | 监护人、作业负责人 | 验收确认签名 |

## 6. 页面导航

| 来源 | 操作 | 目标 |
|------|------|------|
| 作业票详情页 | 点击签名区域/签名按钮 | → 本页面 |
| 本页面 | 确认签名 | → 返回作业票详情页（签名数据回传） |
| 本页面 | 取消/返回 | → 返回作业票详情页（不保存） |

## 8. 交互适配

| 终端 | 适配策略 |
|------|---------|
| 移动端 | 强制横屏，Canvas 占满屏幕宽度，触摸区 ≥44×44px |
| PC 端 | 弹窗模式或内嵌区域，Canvas 400×200px，鼠标绘制 |
| 作业人 | 额外增大线条粗细（3px → 5px），适配手套操作 |

**技术要点**:
- Canvas API 绑定 `mousedown/mousemove/mouseup` + `touchstart/touchmove/touchend`
- 导出为 Base64 PNG（`canvas.toDataURL('image/png')`）
- 移动端使用 `screen.orientation.lock('landscape')` 锁定横屏
- 签名数据压缩后存储，典型大小 5-15KB
