# 05 - 管理后台 - 系统概览

> **路由**: `/admin` | **可见角色**: 仅系统管理员 | **关联组件**: —（ECharts 图表）
> **关联PRD**: [03-统一界面设计.md](../03-统一界面设计.md) §7.1
> **AI 集成**: 5 个 AI 智能体服务状态监控

---

## 1. 页面定位

管理后台首页，系统管理员的工作台。展示系统运行指标、作业分布和快捷操作入口。

## 2. 功能模块

| 模块 | 说明 |
|------|------|
| 顶部导航 | 管理后台 Tab 导航（首页/模板/工作流/权限/人员/数据/审计） |
| 统计卡片 | 今日作业票数、进行中数、在线用户数、系统告警数、🤖 AI 服务状态 |
| 作业分布地图 | 各区域当前作业分布（ECharts 地图/热力图） |
| 本周趋势图 | 7天各类型作业量趋势（ECharts 折线图） |
| 快捷操作 | 6个功能入口卡片 |

## 4. 角色差异

本页面仅管理员可见，无角色差异。

## 5. 状态驱动 UI 变化

不受作业票状态驱动。统计数据为全局汇总。

## 6. 页面导航

| 操作 | 目标 |
|------|------|
| 点击快捷操作卡片 | → 对应管理页面 |
| 点击 Tab 导航 | → 对应管理页面 |
| 点击统计卡片 | → 数据分析页面（带筛选条件） |

## 7. Mock 数据接口

```typescript
// 获取系统概览数据
GET /api/admin/overview
Response: {
  todayTickets: number,
  inProgress: number,
  onlineUsers: number,
  alerts: number,
  aiServices: {                          // 🤖 AI 服务状态
    total: number,                       // 总数（5）
    online: number,                      // 在线数
    services: {
      name: string,                      // Auditor / Spatial-Temporal / DataExchange / KG / MultiExpert
      status: 'online' | 'offline' | 'degraded',
      latency_ms: number                 // 平均响应时间
    }[]
  },
  mapData: { zone: string, count: number, type: string }[],
  trendData: { date: string, type: string, count: number }[]
}
```

## 8. 交互适配

仅 PC 端，响应式布局：
- 宽屏（≥1440px）：统计卡片4列 + 地图/趋势图并排
- 标准（1024-1440px）：统计卡片4列 + 地图/趋势图并排（缩小）
- 窄屏（<1024px）：统计卡片2列 + 地图/趋势图上下排列

## 9. Demo 简化说明

- 统计数据为静态 Mock，不实时计算
- 地图用简化的区域色块替代真实地图
- 趋势图用 ECharts 渲染预置数据
- 🤖 AI 服务状态为 Mock 数据，默认全部在线；点击卡片可展开查看 5 个智能体的名称、状态和响应时间
