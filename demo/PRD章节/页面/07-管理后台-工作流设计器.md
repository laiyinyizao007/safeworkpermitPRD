# 07 - 管理后台 - 工作流设计器

> **路由**: `/admin/workflow` | **可见角色**: 仅系统管理员 | **关联组件**: —
> **关联PRD**: [03-统一界面设计.md](../03-统一界面设计.md) §7.3
> **AI 集成**: Auditor Agent（AI 审计通过作为转换条件）

---

## 1. 页面定位

可视化审批流程编辑器。管理员通过拖拽节点和连线设计作业票的状态流转，点击连线配置转换条件。

## 2. 功能模块

| 模块 | 说明 |
|------|------|
| 流程图画布 | 可视化状态机，节点可拖拽，连线可编辑 |
| 转换配置面板 | 点击连线后显示：转换名称、触发角色、前置条件、触发动作 |
| 工具栏 | 验证/预览/发布按钮 |

## 4. 角色差异

本页面仅管理员可见，无角色差异。

## 5. 状态驱动 UI 变化

不受作业票状态驱动。展示的是工作流定义（元数据），不是某张具体票的状态。

## 6. 页面导航

| 来源 | 操作 | 目标 |
|------|------|------|
| 系统概览 | 点击 [工作流配置] | → 本页面（流程列表） |
| 流程列表 | 点击流程 | → 本页面（编辑模式） |

## 7. Mock 数据接口

```typescript
// 获取工作流定义
GET /api/admin/workflows/:type
Response: {
  states: { name: string, label: string, position: { x, y } }[],
  transitions: { from: string, to: string, label: string, role: string, conditions: string[] }[]
}
```

## 8. 交互适配

仅 PC 端。画布支持缩放（滚轮）和平移（拖拽空白区域）。

## 9. Demo 简化说明

- 展示可视化编辑交互，节点可拖拽移动
- 连线点击可弹出配置面板
- 不实现真实发布，点击发布后 Toast 提示
- 预置动火作业的标准工作流
- 🤖 AI 审计通过条件为 Mock 实现，勾选后在转换配置中显示分数阈值输入框（默认 80 分）
