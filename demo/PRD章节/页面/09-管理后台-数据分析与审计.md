# 09 - 管理后台 - 数据分析与审计

> **路由**: `/admin/analytics` + `/admin/audit` | **可见角色**: 仅系统管理员（安全负责人/审批人可见部分看板） | **关联组件**: —（ECharts 图表）
> **关联PRD**: [03-统一界面设计.md](../03-统一界面设计.md) §7.5
> **AI 集成**: Data Exchange Agent（数据推送监控看板）

---

## 1. 页面定位

合并数据分析看板和审计日志两个功能模块。Demo 中均使用静态 Mock 数据，合并为一个页面文件便于管理。

## 2. 功能模块

### 2.1 数据分析看板（`/admin/analytics`）

| 模块 | 说明 |
|------|------|
| 统计卡片 | 本月作业总数、通过率、平均审批时长、证书预警数 |
| 作业类型分布 | 饼图：8大作业类型占比 |
| 作业趋势 | 折线图：近30天各类型作业量 |
| 区域热力图 | 各区域作业密度 |
| 审批效率分析 | 柱状图：各审批环节平均耗时 |
| 证书预警列表 | 即将过期的人员证书 |

### 2.2 审计日志（`/admin/audit`）

| 模块 | 说明 |
|------|------|
| 多维筛选 | 时间范围、操作人、操作类型、票号 |
| 日志列表 | 时间、操作人、操作类型、目标对象、详情摘要 |
| 详情展开 | 点击展开完整操作详情（变更前后对比） |
| 导出 | CSV 导出按钮 |

## 4. 角色差异

| 角色 | 可见内容 |
|------|---------|
| 系统管理员 | 全部看板 + 审计日志 |
| 安全负责人 | 审核相关看板（审核效率、待审核统计） |
| 审批人 | 审批相关看板（审批效率、待审批统计） |

## 5. 页面导航

| 来源 | 操作 | 目标 |
|------|------|------|
| 系统概览 | 点击 [数据导出] 或 [审计日志] | → 本页面 |
| Tab 导航 | 数据/审计 Tab 切换 | → 对应子页面 |

## 7. 🤖 AI 数据推送监控（Data Exchange Agent）

### 7.3 Mock 数据接口

```typescript
// 获取数据推送监控看板
GET /api/data-exchange/dashboard
Response: {
  summary: {
    todayPushCount: number,       // 今日推送次数
    successRate: number,          // 成功率 (%)
    avgLatency: number,           // 平均延迟 (ms)
    pendingRetry: number          // 待重试条数
  },
  targets: {
    name: string,                 // 推送目标名称
    status: 'online' | 'degraded' | 'offline',
    lastPushTime: string,         // 最近推送时间
    latency: number,              // 延迟 (ms)
    errorMsg?: string             // 失败原因
  }[],
  trendData: {
    date: string,
    success: number,
    failed: number,
    retry: number
  }[]
}
// Mock: 返回预置 4 个推送目标，3 个在线 + 1 个离线
```

## 8. Demo 简化说明
