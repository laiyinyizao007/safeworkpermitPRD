# 节点驱动架构 Demo 集成方案

## 1. 方案概述

本方案说明如何将节点驱动工作流引擎集成到现有 Vue3 MVP Demo 中，实现节点化架构的可视化展示。

### 1.1 集成目标

- 在 Demo 中展示四层节点架构
- 可视化节点执行流程
- 保留现有元数据驱动表单功能
- 实现节点与表单的双向绑定

### 1.2 技术栈

- Vue 3.4 + TypeScript
- Pinia（状态管理）
- jexl（表达式引擎）
- 现有配置文件（7个JSON）

## 2. 目录结构

```
demo/
├── config/
│   ├── nodeLibrary.json          # 节点库配置（已创建）
│   ├── hot_work_metadata.json    # 表单字段定义（已存在）
│   ├── constraintRules.json      # 约束规则（已存在）
│   ├── stateConstraints.json     # 状态约束（已存在）
│   ├── roleViewConfig.json       # 角色视图（已存在）
│   ├── mobileLayoutConfig.json   # 移动端布局（已存在）
│   └── desktopLayoutConfig.json  # 桌面端布局（已存在）
├── src/
│   ├── engine/
│   │   ├── NodeExecutor.ts       # 节点执行引擎（已创建）
│   │   ├── RuleEngine.ts         # 规则引擎（已创建）
│   │   └── PluginLoader.ts       # 插件加载器（已创建）
│   ├── components/
│   │   ├── NodeGraph.vue         # 节点图组件（已创建）
│   │   ├── NodeExecutionMonitor.vue  # 执行监控（已创建）
│   │   └── FormRenderer/         # 表单渲染器（待集成）
│   └── stores/
│       └── workflow.ts           # 工作流状态管理（待创建）
└── docs/
    └── integration/              # 集成方案文档（本文件夹）
        ├── 01-总体架构.md
        ├── 02-核心引擎集成.md
        ├── 03-组件开发.md
        ├── 04-状态管理.md
        └── 05-测试验证.md
```

## 3. 集成步骤概览

### Phase 1: 状态管理层
创建 Pinia store 管理节点执行状态

### Phase 2: 表单渲染器集成
将节点执行引擎集成到现有表单渲染器

### Phase 3: 可视化组件
开发节点图和执行监控界面

### Phase 4: 测试验证
端到端测试节点化工作流
