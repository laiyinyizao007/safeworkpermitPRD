# 02 - 核心引擎集成

## 2.1 NodeExecutor 集成

### 2.1.1 初始化引擎

在 Vue 应用启动时加载节点库配置：

```typescript
// src/main.ts
import { NodeExecutor } from './engine/NodeExecutor';
import nodeLibraryConfig from '../config/nodeLibrary.json';

const app = createApp(App);

// 初始化节点执行引擎
const executor = new NodeExecutor({});
await executor.loadNodeLibrary(nodeLibraryConfig);

app.provide('nodeExecutor', executor);
```

### 2.1.2 在组件中使用

```typescript
// 在表单组件中注入
const executor = inject<NodeExecutor>('nodeExecutor');

// 执行节点
await executor.executeNode('n1'); // 填写-表单
```

## 2.2 RuleEngine 集成

### 2.2.1 审批级别自动匹配

```typescript
import { RuleEngine } from './engine/RuleEngine';

const ruleEngine = new RuleEngine();

// 根据作业级别自动匹配审批级别
const approvalLevel = await ruleEngine.matchApprovalLevel(
  formData.work_level
);

// 结果：special -> 3, level1 -> 2, level2 -> 1
```

### 2.2.2 集成到表单提交流程

```typescript
// 表单提交时自动执行规则
async function submitForm() {
  const level = await ruleEngine.matchApprovalLevel(formData.work_level);
  formData.approval_level = level;

  // 继续提交流程
  await api.submitWorkPermit(formData);
}
```

## 2.3 PluginLoader 集成

### 2.3.1 插件注入逻辑

```typescript
import { PluginLoader } from './engine/PluginLoader';

const pluginLoader = new PluginLoader();

// 检查是否需要注入插件
if (formData.work_level === 'special' &&
    formData.work_zone.includes('罐区')) {

  // 加载并执行插件
  await pluginLoader.executePlugin(
    '@ehs/tank-area-hotwork',
    { data: formData }
  );
}
```

## 2.4 与现有配置文件的关系

### 2.4.1 配置文件映射

| 现有配置 | 节点引擎使用方式 |
|---------|----------------|
| hot_work_metadata.json | FormFieldNode 引用 formSchema |
| constraintRules.json | RuleNode 使用 fieldLevelConstraints |
| stateConstraints.json | 控制节点在不同状态的可执行性 |
| roleViewConfig.json | 控制不同角色可执行的节点 |

### 2.4.2 配置引用示例

```json
{
  "id": "填写-表单",
  "type": "FormFieldNode",
  "config": {
    "formSchemaRef": "hot_work_metadata.json#formSchema"
  }
}
```

引擎会自动加载 `hot_work_metadata.json` 中的字段定义。
