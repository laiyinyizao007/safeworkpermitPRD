# 05 - 通用底座功能需求

## 5.1 基础信息管理

### 5.1.1 组织架构管理

**功能描述：**
支持多层级组织架构配置，包括公司、分厂、车间、班组等层级。

**核心功能：**
- 组织树形结构维护（增删改查）
- 组织层级关系配置（最多支持10级）
- 组织负责人设置
- 组织权限继承规则

**数据模型：**
```
Organization {
  id: string
  name: string
  code: string
  parentId: string
  level: number
  leaderId: string
  status: enum(active, inactive)
}
```

### 5.1.2 区域位置管理

**功能描述：**
管理企业内部作业区域，支持区域分级、坐标定位、风险等级标注。

**核心功能：**
- 区域信息维护（名称、编码、坐标、面积）
- 区域分类（生产区、储罐区、装卸区、办公区等）
- 风险等级标注（高危区、一般区、安全区）
- 区域平面图上传与标注
- 区域边界坐标配置（支持多边形）

**数据模型：**
```
Area {
  id: string
  name: string
  code: string
  type: enum
  riskLevel: enum(high, medium, low)
  boundary: GeoJSON
  floorPlan: string (图片URL)
}
```

### 5.1.3 设备设施管理

**功能描述：**
管理企业内部设备设施台账，支持设备分类、状态跟踪、维保记录。

**核心功能：**
- 设备台账维护（名称、型号、位置、状态）
- 设备分类（反应釜、储罐、泵、阀门等）
- 设备状态管理（运行、停机、检修、报废）
- 设备维保记录
- 设备关联作业历史

**数据模型：**
```
Equipment {
  id: string
  name: string
  code: string
  type: enum
  location: string (区域ID)
  status: enum
  lastMaintenanceDate: date
}
```

## 5.1.4 节点库管理

**功能描述：**
管理工作流节点库，支持四层节点架构的配置、复用和插件化部署。

**核心功能：**
- 节点定义管理（节点ID、类型、配置、复用层级）
- 节点图配置（节点连接、条件跳转、插件注入）
- 节点执行引擎（上下文管理、事件总线、规则引擎）
- 插件管理（插件加载、卸载、版本控制）

**四层节点架构：**

| 层级 | 名称 | 复用范围 | 示例节点 |
|------|------|---------|---------|
| Layer 1 | 通用底座 | 跨行业 | 填写-表单、审批-通用、签名-电子 |
| Layer 2 | 行业无关业务 | 跨行业业务 | 匹配-审批级别、检测-冲突、通知-相关方 |
| Layer 3 | EHS行业特定 | EHS行业 | 环境准入-闸门、验证-资质证书、执行-动火作业 |
| Layer 4 | 微场景插件 | 特定场景 | plugin-罐区动火、plugin-装置区动火 |

**节点配置示例：**
```json
{
  "id": "环境准入-闸门",
  "type": "EnvironmentGateNode",
  "layer": 3,
  "config": {
    "deviceType": "gas_detector",
    "parameters": ["combustible_gas", "oxygen"],
    "thresholds": {
      "combustible_gas": { "max": 0.5, "unit": "%LEL" },
      "oxygen": { "min": 19.5, "max": 23.5, "unit": "%" }
    }
  }
}
```

**规则引擎配置：**
- 审批级别自动匹配（基于作业级别）
- 冲突检测规则（时间、空间、资源）
- 证书验证规则（有效期、适用范围）

**插件注入机制：**
```json
{
  "condition": "data.work_level == 'special' && data.work_zone.includes('罐区')",
  "injectAfter": "环境准入-闸门",
  "pluginNodeRef": "plugin-罐区动火"
}
```

## 5.3 JSA风险库

### 5.3.1 风险库管理

**功能描述：**
建立标准化风险知识库，支持风险分类、等级评估、控制措施配置。

**核心功能：**
- 风险项维护（风险描述、类型、等级）
- 风险分类（火灾爆炸、中毒窒息、高处坠落等）
- 风险等级评估（LEC法：L×E×C）
- 控制措施配置（工程措施、管理措施、PPE）
- 风险库导入导出（Excel模板）

**数据模型：**
```
RiskItem {
  id: string
  name: string
  category: enum
  description: string
  likelihood: number (L)
  exposure: number (E)
  consequence: number (C)
  riskLevel: number (L×E×C)
  controlMeasures: array
}
```

### 5.3.2 JSA模板管理

**功能描述：**
为不同作业类型配置JSA分析模板，支持模板复用、快速填写。

**核心功能：**
- JSA模板配置（作业步骤、风险点、控制措施）
- 模板分类（按作业类型）
- 模板版本管理
- 模板引用统计
- 模板效果评估

**数据模型：**
```
JSATemplate {
  id: string
  workType: enum
  steps: array [
    {
      stepNo: number
      description: string
      risks: array
      controls: array
    }
  ]
  version: string
  usageCount: number
}
```

### 5.3.3 智能风险推荐

**功能描述：**
基于作业类型、区域、设备等信息，智能推荐相关风险点和控制措施。

**核心功能：**
- 基于规则的风险匹配
- 历史作业风险学习
- 风险推荐排序（按相关度）
- 推荐准确率统计
- 用户反馈收集

**推荐算法：**
```
推荐得分 = 作业类型匹配度 × 0.4
         + 区域风险匹配度 × 0.3
         + 历史频率 × 0.2
         + 季节因素 × 0.1
```

## 5.4 IoT接口层

### 5.4.1 设备接入管理

**功能描述：**
支持多种IoT设备接入，提供统一设备管理界面。

**核心功能：**
- 设备注册（设备ID、类型、协议）
- 设备认证（证书、密钥）
- 设备状态监控（在线、离线、故障）
- 设备配置下发
- 设备固件升级

**支持设备类型：**
- 气体检测仪（可燃气体、有毒气体、氧气）
- 视频监控（摄像头、NVR）
- 定位设备（UWB标签、蓝牙信标）
- 智能穿戴（安全帽、手环）
- 环境监测（温湿度、风速风向）

### 5.4.2 数据采集与处理

**功能描述：**
实时采集IoT设备数据，进行清洗、转换、存储。

**核心功能：**
- 数据采集（MQTT/HTTP订阅）
- 数据清洗（异常值过滤）
- 数据转换（单位换算、格式统一）
- 数据存储（InfluxDB时序库）
- 数据查询（按时间、设备、指标）

**数据流：**
```
IoT设备 → MQTT Broker → 数据清洗 → InfluxDB → API查询
```

### 5.4.3 告警规则配置

**功能描述：**
配置IoT设备告警规则，支持多级告警、自动通知。

**核心功能：**
- 告警规则配置（阈值、持续时间、告警级别）
- 告警通知配置（短信、邮件、APP推送）
- 告警升级机制（超时未处理自动升级）
- 告警屏蔽（维护期间）
- 告警统计分析

**告警级别：**
- 紧急（红色）：立即处理，自动停止作业
- 重要（橙色）：30分钟内处理
- 一般（黄色）：2小时内处理
- 提示（蓝色）：仅记录

## 5.6 SIMOPs冲突检测

详见：[docs/architecture/simops-algorithm.md](../docs/architecture/simops-algorithm.md)

### 5.6.1 空间冲突检测

**功能描述：**
检测多个作业的空间范围是否重叠或过近。

**检测规则：**
- 作业区域完全重叠 → 高风险冲突
- 作业区域部分重叠 → 中风险冲突
- 作业区域距离<安全距离 → 低风险冲突

**安全距离标准：**
- 动火作业：30米
- 受限空间作业：10米
- 高处作业：垂直5米、水平10米
- 吊装作业：吊装半径+10米

### 5.6.2 时间冲突检测

**功能描述：**
检测多个作业的时间段是否重叠。

**检测规则：**
- 时间完全重叠 → 高风险
- 时间部分重叠 → 中风险
- 时间相邻（间隔<1小时）→ 低风险

### 5.6.3 逻辑冲突检测

**功能描述：**
基于业务规则检测作业之间的逻辑冲突。

**冲突规则库：**
- 动火作业 + 受限空间作业（相邻区域）→ 可燃气体泄漏风险
- 高处作业 + 地面作业（垂直重叠）→ 坠物伤人风险
- 临时用电 + 动火作业（同一区域）→ 电气火花引燃风险
- 盲板抽堵 + 动火作业（同一管线）→ 介质泄漏风险

## 5.7 审批引擎

### 5.7.1 审批流程配置

**功能描述：**
灵活配置审批流程，支持多级审批、条件审批、会签/或签。

**核心功能：**
- 审批节点配置（审批人、审批角色）
- 审批条件配置（按风险等级、作业类型）
- 审批方式配置（会签、或签、依次审批）
- 审批时限配置（超时自动升级）
- 审批流程版本管理

**审批流程示例：**
```
一级作业（高风险）：
  班组长 → 车间主任 → 安全科 → 分管领导

二级作业（中风险）：
  班组长 → 车间主任 → 安全科

三级作业（低风险）：
  班组长 → 车间主任
```

### 5.7.2 电子签名

**功能描述：**
支持CA电子签名，确保审批法律效力。

**核心功能：**
- CA证书集成（对接第三方CA机构）
- 签名验证（RSA-2048 / SM2国密）
- 时间戳服务（RFC 3161标准）
- 签名可视化（显示签名图章）
- 签名证书链验证

### 5.7.3 审批记录追溯

**功能描述：**
完整记录审批过程，支持审批历史查询、责任追溯。

**核心功能：**
- 审批日志记录（时间、审批人、意见、IP）
- 审批历史查询
- 审批统计分析（通过率、平均时长）
- 审批责任追溯
- 审批数据导出

## 5.9 元数据表单引擎（G-01）

详见：[功能需求积木化拆解方案 §2.1 G-01](../功能需求_积木化拆解方案.md)、[G-01 配置 Schema 规范](../分析内容/code/功能需求%20拆解/G01-元数据表单引擎配置Schema规范.md)

### 5.9.1 核心能力

**功能描述：**
基于 JSONB 的元数据驱动表单服务，支持零代码配置不同企业的作业票字段、布局和业务逻辑。

**核心功能：**
- 可视化表单设计器（拖拽式 Low-Code Builder）
- 8 种作业票标准模板库
- 表达式引擎（jexl/json-logic）
- 多级覆盖机制（系统默认 → 公司定制 → 项目定制）
- 状态机驱动（Draft → Verify → Executing → Closed）
- 约束组件（字段级、字段间依赖、状态约束、硬约束）

### 5.9.2 表单设计器

**Low-Code Builder 界面**：
- 左侧：组件库（20-30 个标准组件）
- 中间：画布区（拖拽编排）
- 右侧：属性面板（配置字段属性、校验规则、联动逻辑）
- 底部：预览区（实时渲染手机/PC 端效果）

**AI Copilot 辅助**：
- 自然语言描述生成元数据变更
- 示例："帮我增加一个'施工人员签名'字段，并在'危险系数'大于 3 时显示"
- 自动生成对应的 Schema + Layout JSON

### 5.9.3 模板库管理

**8 种标准模板**：

| 模板名称 | 配置包文件 | 特有字段 |
|---------|-----------|---------|
| 动火作业票 | hot_work_config | 动火方式、可燃气体检测 |
| 受限空间作业票 | confined_space_config | 通风时间、多气体检测 |
| 盲板抽堵作业票 | blind_plate_config | 盲板编号、介质信息 |
| 高处作业票 | height_work_config | 作业高度、天气条件 |
| 吊装作业票 | lifting_config | 吊装重量、起重设备 |
| 临时用电作业票 | temp_power_config | 电压/功率、漏电保护 |
| 动土作业票 | excavation_config | 动土深度、地下管线 |
| 断路作业票 | road_block_config | 断路位置、绕行方案 |

**模板结构**：
- form_schema.json（字段定义）
- layout_config.json（布局配置）
- constraint_rules.json（约束规则）
- role_view_config.json（角色视图配置）

### 5.9.4 组件库

**四大组件类别**：

| 类别 | 组件 | 用途 |
|------|------|------|
| **基础输入** | Text/Textarea、Number、Radio/Checkbox/Select、Date/Time Picker | 构建表单基石 |
| **业务逻辑** | 电子签名、人员/部门选择器、地理位置、附件/图片上传 | 解决差异化流程 |
| **结构组件** | 栅格/行、卡片/分组、动态明细表 | UI 布局可配置 |
| **逻辑控制** | 计算字段、隐藏域 | 自动计算与系统字段 |

**MVP 组件数量**：8-10 个（基础输入 + 签名 + 图片上传）
**成熟版组件数量**：20-30 个（含关联设备台账、扫码识别、多级联动）

### 5.9.5 表达式引擎

**支持的表达式类型**：
- 条件显示：`visibleIf: "data.work_type == 'hot_work'"`
- 条件启用：`enabledIf: "data.gas_detection_passed"`
- 计算字段：`value: "data.L * data.E * data.C"`
- 校验规则：`validate: "data.oxygen >= 19.5 && data.oxygen <= 23.5"`

**表达式语法**：
- 使用 jexl 或 json-logic
- 支持逻辑运算（&&、||、!）
- 支持比较运算（==、!=、>、<、>=、<=）
- 支持算术运算（+、-、*、/）

### 5.9.6 约束组件

**字段级约束**：

```json
{
  "key": "oxygen_level",
  "type": "number",
  "label": "氧气浓度 (%)",
  "required": true,
  "min": 19.5,
  "max": 23.5,
  "errorMessage": "氧气浓度必须在 19.5-23.5% 之间"
}
```

**字段间依赖约束**：

```json
{
  "key": "safety_belt_type",
  "type": "select",
  "label": "安全带类型",
  "visibleIf": "data.height > 2",
  "options": ["全身式", "半身式"]
}
```

**状态约束**：

```json
{
  "key": "gas_detection_value",
  "type": "number",
  "label": "气体检测值",
  "readonlyIf": "state != 'Verify'",
  "requiredIf": "state == 'Verify'"
}
```

**硬约束（不可绕过）**：

```json
{
  "key": "site_photo",
  "type": "image_upload",
  "label": "现场照片",
  "required": true,
  "props": {
    "source": "camera_only",
    "minCount": 3,
    "maxCount": 9,
    "aiReview": true
  }
}
```

### 5.9.8 工作流集成

**Design Phase（设计阶段）**：
1. 选择模板（8 种标准模板）
2. 拖拽组件（20-30 个标准组件）
3. 配置属性（字段属性、校验规则、联动逻辑）
4. 预览效果（手机/PC 端实时渲染）

**Deployment Phase（部署阶段）**：
1. 版本管理（v1.0、v1.1、v2.0）
2. 灰度发布（先给部分分公司试用）
3. 权限分配（施工方填写、安全员审批）

**Execution Phase（执行阶段）**：
1. 扫码拉取（根据 company_id + work_type 加载元数据）
2. 动态渲染（表单渲染引擎生成 UI）
3. 校验拦截（必填项、取值范围、联动逻辑）
4. 数据沉淀（JSONB 存储 + PDF 归档）
