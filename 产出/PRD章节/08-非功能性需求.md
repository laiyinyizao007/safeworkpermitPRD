# 08 - 非功能性需求

## 8.1 性能需求

### 8.1.1 响应时间

| 操作类型 | 目标响应时间（P95） | 说明 |
|---------|-------------------|------|
| **页面加载** | ≤2秒 | 首屏加载时间 |
| **API接口** | ≤200ms | 业务接口响应 |
| **数据库查询** | ≤100ms | 单次查询 |
| **文件上传** | ≥1MB/s | 图片、PDF上传 |
| **地图渲染** | ≤1秒 | 作业分布地图 |
| **报表生成** | ≤30秒 | 复杂统计报表 |

### 8.1.2 并发性能

| 指标 | 目标值 | 测试场景 |
|------|--------|---------|
| **并发用户数** | ≥5000 | 全厂同时在线 |
| **并发作业票** | ≥1000 | 同时处理作业票 |
| **TPS** | ≥500 | 每秒事务处理数 |
| **QPS** | ≥2000 | 每秒查询数 |

### 8.1.3 数据处理能力

| 指标 | 目标值 | 说明 |
|------|--------|------|
| **日作业票处理量** | ≥10万张 | 支持大型企业 |
| **IoT数据采集** | ≥1000设备 | 每秒10000条数据 |
| **时序数据存储** | ≥1年 | InfluxDB存储 |
| **文件存储** | ≥10TB | MinIO存储 |

## 8.2 可用性需求

### 8.2.1 系统可用性

| 指标 | 目标值 | 计算方式 |
|------|--------|---------|
| **系统可用性** | ≥99.5% | 月度统计 |
| **计划内停机** | ≤4小时/月 | 维护窗口 |
| **计划外停机** | ≤2小时/月 | 故障时间 |
| **MTBF** | ≥720小时 | 平均故障间隔 |
| **MTTR** | ≤30分钟 | 平均恢复时间 |

### 8.2.2 容错能力

**单点故障消除：**
- 应用服务器：至少2台，负载均衡
- 数据库：主从复制，自动故障切换
- Redis：主从+哨兵模式
- 消息队列：集群部署

**故障恢复：**
- 自动故障检测（健康检查）
- 自动故障切换（≤30秒）
- 数据自动备份（每日全量+实时增量）
- 灾难恢复演练（每季度1次）

### 8.2.3 降级策略

**降级场景：**

| 场景 | 降级策略 | 影响 |
|------|---------|------|
| **数据库压力过大** | 关闭非核心查询，只保留作业票CRUD | 统计报表暂时不可用 |
| **IoT设备离线** | 使用缓存数据，手工录入 | 实时监控暂时不可用 |
| **第三方服务故障** | 跳过集成，核心流程继续 | 部分功能受限 |
| **网络中断** | 启用离线模式 | 数据延迟同步 |

## 8.3 安全性需求

### 8.3.1 身份认证

**认证方式：**
- 用户名+密码（强密码策略）
- 短信验证码（登录、敏感操作）
- 企业微信/钉钉单点登录（SSO）
- CA证书（电子签名）

**密码策略：**
- 长度≥8位
- 包含大小写字母、数字、特殊字符
- 90天强制修改
- 禁止使用近3次历史密码
- 连续5次错误锁定账号

### 8.3.2 权限控制

**权限模型：**
- RBAC（基于角色）：预定义角色（管理员、审批人、作业人等）
- ABAC（基于属性）：动态权限（只能审批本部门作业票）

**权限粒度：**
- 功能权限（菜单、按钮）
- 数据权限（本人、本部门、全部）
- 操作权限（查看、新增、修改、删除、审批）

**特殊权限：**
- 超级管理员：拥有所有权限，操作需二次确认
- 审计员：只读权限，查看所有数据
- 应急账号：紧急情况下临时开通，24小时后自动失效

### 8.3.3 数据加密

**传输加密：**
- HTTPS/TLS 1.3
- 证书：2048位RSA或ECC
- 禁用弱加密算法（SSL 3.0、TLS 1.0/1.1）

**存储加密：**
- 敏感字段加密（AES-128）：身份证号、手机号、证书编号
- 密码加密：BCrypt + Salt
- 文件加密：MinIO服务端加密

**国密支持：**
- SM2（非对称加密）
- SM3（哈希算法）
- SM4（对称加密）

### 8.3.4 审计日志

**日志内容：**
- 用户登录/登出
- 作业票创建/修改/删除
- 审批操作
- 权限变更
- 系统配置修改
- 数据导出

**日志字段：**
- 操作时间
- 操作人（用户ID、姓名、IP）
- 操作类型
- 操作对象（作业票ID、用户ID等）
- 操作结果（成功/失败）
- 详细信息（修改前后对比）

**日志保留：**
- 在线日志：6个月
- 归档日志：3年
- 重要日志：永久保存

### 8.3.5 安全防护

**防护措施：**
- SQL注入防护（参数化查询）
- XSS防护（输入过滤、输出转义）
- CSRF防护（Token验证）
- 文件上传防护（类型检查、大小限制、病毒扫描）
- API限流（1000次/分钟/IP）
- DDoS防护（云WAF）

**安全扫描：**
- 代码静态扫描（SonarQube）
- 依赖漏洞扫描（OWASP Dependency Check）
- 渗透测试（每季度1次）
- 安全加固（定期更新补丁）

## 8.5 可维护性需求

### 8.5.1 代码质量

| 指标 | 目标值 | 工具 |
|------|--------|------|
| **测试覆盖率** | ≥85% | JaCoCo |
| **代码复杂度** | ≤10 | SonarQube |
| **代码重复率** | ≤5% | SonarQube |
| **代码规范** | 0个严重违规 | Checkstyle |

### 8.5.2 日志规范

**日志级别：**
- ERROR：系统错误，需立即处理
- WARN：警告信息，需关注
- INFO：关键业务操作
- DEBUG：调试信息（生产环境关闭）

**日志格式：**
```
[时间] [级别] [线程] [类名] [TraceId] - 日志内容
```

**日志存储：**
- 应用日志：ELK Stack
- 访问日志：Nginx日志
- 数据库日志：慢查询日志
- 审计日志：独立数据库表

### 8.5.3 监控告警

**监控指标：**
- 系统指标：CPU、内存、磁盘、网络
- 应用指标：QPS、响应时间、错误率
- 业务指标：作业票数量、审批通过率
- 数据库指标：连接数、慢查询、锁等待

**告警规则：**
- CPU使用率>80%持续5分钟
- 内存使用率>85%
- 磁盘使用率>90%
- 接口错误率>5%
- 数据库连接数>80%

**告警通知：**
- 短信（紧急告警）
- 邮件（一般告警）
- 企业微信/钉钉（实时告警）
- 电话（严重告警，5分钟未处理）
