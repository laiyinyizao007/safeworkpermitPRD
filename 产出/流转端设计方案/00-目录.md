## 文档概述

本设计方案详细描述作业票系统**流转端**的完整设计。流转端是配置端的对称互补——配置端负责"定义规则、生产元数据"，流转端负责"消费元数据、驱动任务全生命周期执行"。

### 配置端 vs 流转端

```mermaid
flowchart LR
    subgraph ConfigEnd["配置端 (定义规则)"]
        C1[元数据Schema定义]
        C2[组件库管理]
        C3[约束规则配置]
        C4[状态机模板定义]
    end

    subgraph RuntimeEnd["流转端 (执行实例)"]
        R1[任务创建与编排]
        R2[审批流程引擎]
        R3[现场执行监控]
        R4[待办与任务分发]
    end

    ConfigEnd -->|元数据配置包<br/>版本化发布| RuntimeEnd
    RuntimeEnd -->|执行数据反馈<br/>AI训练样本| ConfigEnd
```

### 核心设计理念

- **事件驱动**: 节点完成 → Event → 状态机决定 Next State
- **主表+子表**: Task 容器挂载多个 Permit，流转时可针对部分作业表锁定权限
- **AI纠偏**: AI 生成流程渲染到低代码画布，用户可手动拖拽调整
- **三维权限**: 状态 × 角色 × 字段 = 最终权限，运行时动态计算
