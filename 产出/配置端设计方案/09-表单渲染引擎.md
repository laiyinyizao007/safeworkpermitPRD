# 09 - 表单渲染引擎

> **本章导读**: 本章详细介绍表单渲染引擎的设计，包括元数据解析器、组件注册表、表达式引擎集成、状态机实现和完整渲染流程。

---

## 9.1 渲染引擎概览

### 9.1.1 引擎定位

表单渲染引擎是连接"配置端"与"执行端"的核心桥梁，负责将元数据配置JSON动态渲染为可交互的表单界面。

```mermaid
flowchart LR
    subgraph "输入"
        Meta[元数据JSON]
        Data[实例数据]
        Context[上下文信息<br/>用户/角色/状态]
    end

    subgraph "渲染引擎 Form Renderer"
        Parser[元数据解析器]
        Registry[组件注册表]
        Expression[表达式引擎]
        SM[状态机]
        Validator[校验器]
    end

    subgraph "输出"
        UI[可交互表单UI]
        Events[事件流]
        Output[表单数据]
    end

    Meta --> Parser
    Data --> Parser
    Context --> Parser
    Parser --> Registry --> Expression --> SM --> Validator
    Validator --> UI & Events & Output

    style Parser fill:#e1f5ff
    style Registry fill:#fff4e6
    style Expression fill:#e8f5e9
```

### 9.1.2 设计目标

| 目标 | 指标 | 说明 |
|------|------|------|
| **高性能** | 首屏渲染 < 500ms | 50个字段的表单在移动端流畅渲染 |
| **高兼容** | 支持多端 | PC Web、移动端小程序、打印 |
| **可扩展** | 插件化组件 | 支持自定义组件注册 |
| **强校验** | 实时+提交双校验 | 确保数据质量 |

## 9.2 元数据解析器

### 9.2.1 解析流程

```mermaid
flowchart TD
    Input[元数据JSON] --> Validate[Schema校验<br/>验证JSON结构合法性]
    Validate -->|不合法| Error[抛出解析错误]
    Validate -->|合法| Parse[分层解析]

    Parse --> S1[解析Schema层<br/>提取字段定义]
    Parse --> S2[解析Layout层<br/>提取布局配置]
    Parse --> S3[解析Rules层<br/>提取业务规则]
    Parse --> S4[解析StateMachine<br/>提取状态定义]

    S1 --> Merge[合并与关联]
    S2 --> Merge
    S3 --> Merge
    S4 --> Merge

    Merge --> Override[多级覆盖处理<br/>系统→公司→项目]
    Override --> Tree[生成渲染树<br/>Render Tree]
    Tree --> Output[输出可渲染结构]
```

### 9.2.2 解析器核心接口

```typescript
interface MetadataParser {
  // 解析完整元数据
  parse(metadata: FormMetadata, context: RenderContext): RenderTree;

  // 解析Schema层
  parseSchema(schema: SchemaDefinition): FieldNode[];

  // 解析Layout层
  parseLayout(layout: LayoutDefinition, fields: FieldNode[]): LayoutNode;

  // 解析Rules层
  parseRules(rules: RulesDefinition): RuleBinding[];

  // 多级覆盖合并
  mergeOverrides(base: FormMetadata, overrides: FormMetadata[]): FormMetadata;
}

interface RenderContext {
  currentUser: UserInfo;       // 当前用户
  currentRole: string;         // 当前角色
  currentState: string;        // 当前状态
  permitData: Record<string, any>; // 已有数据
  deviceType: 'pc' | 'mobile'; // 设备类型
}

interface RenderTree {
  root: LayoutNode;            // 布局根节点
  fields: Map<string, FieldNode>; // 字段映射
  rules: RuleBinding[];        // 规则绑定
  stateMachine: StateMachineConfig; // 状态机配置
}
```

### 9.2.3 多级覆盖解析

```mermaid
flowchart TD
    L1[Level 1: 系统默认配置] --> Merge1{合并}
    L2[Level 2: 公司定制配置] --> Merge1
    Merge1 --> Result1[公司级配置]

    Result1 --> Merge2{合并}
    L3[Level 3: 项目定制配置] --> Merge2
    Merge2 --> Final[最终生效配置]

    subgraph "合并规则"
        R1[字段级: 深度合并]
        R2[布局级: 整体替换]
        R3[规则级: 追加合并]
    end
```

**合并策略**:

| 配置层 | 合并方式 | 说明 |
|-------|---------|------|
| Schema字段 | 深度合并(Deep Merge) | 下级可新增字段、修改属性，不可删除上级字段 |
| Layout布局 | 整体替换(Override) | 下级提供完整布局则替换，否则继承上级 |
| Rules规则 | 追加合并(Append) | 下级规则追加到上级规则之后，同名规则覆盖 |
| 状态机 | 不可覆盖 | 状态机定义仅系统级可配置 |

### 9.3.2 组件注册接口

```typescript
interface ComponentRegistry {
  // 注册组件
  register(type: string, component: ComponentDefinition): void;

  // 获取组件
  get(type: string): ComponentDefinition | undefined;

  // 检查组件是否已注册
  has(type: string): boolean;

  // 获取所有已注册组件
  getAll(): Map<string, ComponentDefinition>;

  // 注销组件
  unregister(type: string): void;
}

interface ComponentDefinition {
  type: string;                    // 组件类型标识
  name: string;                    // 组件显示名称
  category: ComponentCategory;     // 组件分类
  component: VueComponent;         // Vue组件实例
  propsSchema: JSONSchema;         // 属性Schema定义
  defaultProps: Record<string, any>; // 默认属性
  validator?: FieldValidator;      // 内置校验器
  formatter?: ValueFormatter;      // 值格式化器
  platforms: ('pc' | 'mobile' | 'print')[]; // 支持平台
}
```

## 9.4 表达式引擎

### 9.4.1 表达式类型

| 表达式类型 | 用途 | 示例 |
|-----------|------|------|
| **条件表达式** | 控制显隐、只读 | `data.fire_level === 'special'` |
| **计算表达式** | 自动计算字段值 | `data.work_hours * data.worker_count` |
| **校验表达式** | 自定义校验规则 | `data.end_time > data.start_time` |
| **格式化表达式** | 显示格式转换 | `FORMAT_DATE(data.created_at, 'YYYY-MM-DD')` |

### 9.4.2 表达式引擎架构

```mermaid
flowchart TD
    Input[表达式字符串] --> Lexer[词法分析器<br/>Tokenize]
    Lexer --> Parser[语法分析器<br/>Parse to AST]
    Parser --> Optimizer[优化器<br/>常量折叠/缓存]
    Optimizer --> Evaluator[求值器<br/>Evaluate]

    Context[上下文数据] --> Evaluator
    Functions[内置函数库] --> Evaluator

    Evaluator --> Result[计算结果]
```

## 9.5 渲染流程

### 9.5.1 完整渲染流程

```mermaid
flowchart TD
    Start[接收渲染请求] --> Fetch[获取元数据配置]
    Fetch --> Cache{缓存命中?}
    Cache -->|是| UseCached[使用缓存配置]
    Cache -->|否| LoadDB[从数据库加载]
    LoadDB --> SetCache[写入缓存]
    SetCache --> UseCached

    UseCached --> Parse[元数据解析]
    Parse --> Override[多级覆盖合并]
    Override --> BuildTree[构建渲染树]

    BuildTree --> ResolveComponents[解析组件<br/>从注册表获取]
    ResolveComponents --> BindData[绑定数据<br/>填充已有值]
    BindData --> EvalExpressions[计算表达式<br/>显隐/只读/计算]
    EvalExpressions --> ApplyState[应用状态机<br/>字段权限控制]
    ApplyState --> Render[渲染UI组件]

    Render --> Mount[挂载到DOM]
    Mount --> Listen[监听用户交互]

    Listen --> Change{数据变更}
    Change --> Recompute[重新计算表达式]
    Recompute --> Validate[实时校验]
    Validate --> UpdateUI[更新UI]
    UpdateUI --> Listen
```

### 9.5.2 增量更新机制

```mermaid
flowchart LR
    Change[字段值变更] --> Deps[依赖分析<br/>哪些字段受影响]
    Deps --> Affected[受影响字段列表]
    Affected --> Recompute[仅重算受影响字段]
    Recompute --> Diff[Diff对比]
    Diff --> Patch[最小化DOM更新]
```

**依赖图示例**:

```
fire_level 变更
  ├── special_fire_section (显隐)
  ├── fire_watch_duration (计算)
  ├── safety_director_sign (必填切换)
  └── risk_score (重新计算)
```

### 9.5.3 渲染性能优化

| 优化策略 | 实现方式 | 效果 |
|---------|---------|------|
| **元数据缓存** | Redis缓存 + 本地缓存双层 | 减少数据库查询 |
| **渲染树缓存** | 相同配置复用渲染树 | 减少解析开销 |
| **懒加载组件** | 非可视区域延迟加载 | 首屏时间减少40% |
| **虚拟滚动** | 长列表虚拟化渲染 | 内存占用减少60% |
| **表达式缓存** | 缓存AST和计算结果 | 重复计算减少80% |
| **批量更新** | 合并多次数据变更 | 减少DOM操作次数 |
| **Web Worker** | 复杂计算移至Worker | 主线程不阻塞 |

## 9.6 多端适配

### 9.6.1 适配策略

```mermaid
flowchart TD
    RenderTree[渲染树] --> Adapter{平台适配器}

    Adapter -->|PC Web| PCRenderer[PC渲染器<br/>Vue 3 + Element Plus]
    Adapter -->|移动端| MobileRenderer[移动端渲染器<br/>uni-app / 小程序]
    Adapter -->|打印| PrintRenderer[打印渲染器<br/>HTML to PDF]

    PCRenderer --> PCLayout[多列布局<br/>侧边栏+主区域]
    MobileRenderer --> MobileLayout[单列布局<br/>卡片式堆叠]
    PrintRenderer --> PrintLayout[A4纸面布局<br/>表格式排版]
```

### 9.6.2 平台差异处理

| 特性 | PC Web | 移动端 | 打印 |
|------|--------|-------|------|
| 布局 | 多列栅格 | 单列堆叠 | 表格排版 |
| 签名 | 鼠标绘制 | 手指绘制 | 显示签名图片 |
| 拍照 | 文件选择 | 调用相机 | 显示照片 |
| 定位 | 浏览器定位 | GPS定位 | 显示位置文本 |
| 交互 | 鼠标+键盘 | 触摸 | 无交互 |
| 离线 | 不支持 | 支持 | N/A |
