# 10 - 特色功能

> **本章导读**: 本章介绍配置端的三大特色功能：AI Copilot辅助配置、强制约束组件和多级覆盖机制。

## 10.1 AI Copilot 辅助配置

### 10.1.1 功能概览

AI Copilot 是配置端的智能助手，帮助配置管理员更高效地完成表单设计。

```mermaid
flowchart TD
    subgraph "AI Copilot 能力矩阵"
        AI[AI Copilot] --> NLP[自然语言生成<br/>Natural Language to Metadata]
        AI --> OCR[纸质表单识别<br/>Paper Form OCR]
        AI --> Recommend[智能推荐<br/>Smart Recommendation]
        AI --> Check[智能检查<br/>Configuration Audit]
    end

    NLP --> N1[描述需求 → 生成字段]
    NLP --> N2[描述规则 → 生成约束]

    OCR --> O1[拍照上传 → 识别字段]
    OCR --> O2[识别布局 → 生成配置]

    Recommend --> R1[推荐常用字段]
    Recommend --> R2[推荐约束规则]
    Recommend --> R3[推荐布局方案]

    Check --> C1[检查配置完整性]
    Check --> C2[检查规则冲突]
    Check --> C3[检查合规性]
```

### 10.1.3 纸质表单OCR识别

**使用场景**: 将企业现有的纸质作业票表单拍照上传，AI自动识别并转换为元数据配置。

```mermaid
flowchart TD
    Upload[上传纸质表单照片] --> Preprocess[图像预处理<br/>去噪/矫正/增强]
    Preprocess --> OCR[OCR文字识别]
    OCR --> Structure[结构化分析<br/>识别表格/区块/字段]

    Structure --> Fields[提取字段信息]
    Structure --> Layout[提取布局结构]
    Structure --> Rules[推断校验规则]

    Fields --> Generate[生成元数据配置]
    Layout --> Generate
    Rules --> Generate

    Generate --> Preview[预览生成结果]
    Preview --> Adjust[人工调整修正]
    Adjust --> Apply[应用到画布]
```

**识别能力**:

| 识别项 | 准确率 | 说明 |
|-------|--------|------|
| 字段标签 | 95%+ | 识别表格中的标签文字 |
| 字段类型 | 85%+ | 根据格式推断（日期、数字、文本） |
| 表格结构 | 90%+ | 识别行列关系和分组 |
| 勾选框 | 90%+ | 识别checkbox类型字段 |
| 签名区域 | 85%+ | 识别签名位置 |

## 10.2 强制约束组件

### 10.2.1 强制约束概览

强制约束组件是系统的安全底线，确保关键安全操作不可被绕过。

```mermaid
flowchart TD
    subgraph "强制约束组件"
        H1[强制拍照<br/>camera_only]
        H2[分步确认<br/>sequential_checklist]
        H3[位置锁定<br/>geo_fencing]
        H4[AI审查<br/>image_recognition]
    end

    H1 --> V1[确保现场真实照片<br/>禁止相册选择]
    H2 --> V2[确保逐项确认<br/>禁止批量勾选]
    H3 --> V3[确保人在现场<br/>禁止远程操作]
    H4 --> V4[确保照片合规<br/>AI自动审查]

    style H1 fill:#ffebee
    style H2 fill:#ffebee
    style H3 fill:#ffebee
    style H4 fill:#ffebee
```

### 10.2.2 强制拍照（camera_only）

**设计目的**: 确保上传的照片是现场实时拍摄，而非从相册选择的历史照片。

**约束规则**:
- 只允许调用设备相机拍照，禁止从相册选择
- 照片自动添加水印（时间、GPS坐标、操作人）
- 照片EXIF信息校验（拍摄时间必须在当前操作时间±5分钟内）
- 照片GPS校验（拍摄位置必须在作业区域范围内）

**配置示例**:

```json
{
  "key": "site_photo",
  "type": "image_upload",
  "label": "现场环境照片",
  "required": true,
  "props": {
    "source": "camera_only",
    "minCount": 2,
    "maxCount": 5,
    "watermark": {
      "enabled": true,
      "content": ["timestamp", "gps", "operator", "permit_no"]
    },
    "validation": {
      "exif_time_tolerance": 300,
      "geo_fence_check": true,
      "min_resolution": "1280x720"
    }
  }
}
```

### 10.2.3 分步确认（sequential_checklist）

**设计目的**: 确保操作人员逐项确认安全措施，防止一次性全部勾选。

**约束规则**:
- 必须按顺序逐项确认，不可跳过
- 每项确认之间有最小间隔时间（默认3秒）
- 确认后不可取消（防止误操作需走撤销流程）
- 记录每项确认的时间戳

```mermaid
flowchart LR
    S1[第1项<br/>灭火器就位] -->|确认 ≥3s| S2[第2项<br/>消防水带就位]
    S2 -->|确认 ≥3s| S3[第3项<br/>通风设备开启]
    S3 -->|确认 ≥3s| S4[第4项<br/>监护人到位]
    S4 -->|确认 ≥3s| S5[全部确认完成]

    style S1 fill:#e8f5e9
    style S5 fill:#e8f5e9
```

**配置示例**:

```json
{
  "key": "safety_checklist",
  "type": "sequential_checklist",
  "label": "安全措施确认清单",
  "required": true,
  "props": {
    "sequential": true,
    "minInterval": 3000,
    "irreversible": true,
    "items": [
      { "key": "fire_extinguisher", "label": "灭火器已就位", "helpText": "确认灭火器在作业点5米范围内" },
      { "key": "fire_hose", "label": "消防水带已连接", "helpText": "确认消防水带已连接到最近消火栓" },
      { "key": "ventilation", "label": "通风设备已开启", "helpText": "确认机械通风设备正常运行" },
      { "key": "supervisor_present", "label": "监护人已到位", "helpText": "确认监护人在现场且可视范围内" }
    ],
    "recordTimestamp": true
  }
}
```

### 10.2.4 位置锁定（geo_fencing）

**设计目的**: 确保关键操作只能在指定的作业区域内执行。

```mermaid
flowchart TD
    Action[用户操作] --> GPS[获取GPS位置]
    GPS --> Check{位置校验}

    Check -->|在围栏内| Allow[允许操作]
    Check -->|在围栏外| Block[阻止操作]

    Block --> Alert[提示: 请在作业区域内操作]
    Block --> Log[记录违规尝试]

    subgraph "围栏类型"
        Circle[圆形围栏<br/>中心点+半径]
        Polygon[多边形围栏<br/>多个坐标点]
        Building[建筑围栏<br/>楼层+区域]
    end
```

**配置示例**:

```json
{
  "geo_fencing": {
    "enabled": true,
    "fence_type": "circle",
    "center": { "lat": 31.2304, "lng": 121.4737 },
    "radius": 200,
    "unit": "meters",
    "enforced_actions": ["submit", "sign", "photo", "gas_detection"],
    "tolerance": 10,
    "offline_behavior": "warn_and_allow",
    "violation_action": {
      "block": true,
      "notify": ["supervisor", "safety_officer"],
      "log": true
    }
  }
}
```

### 10.2.5 AI审查（image_recognition）

**设计目的**: 利用AI自动审查上传的照片是否符合安全要求。

**审查能力**:

| 审查项 | 说明 | 准确率 |
|-------|------|--------|
| PPE检测 | 检测安全帽、安全带、防护服 | 92%+ |
| 环境检测 | 检测明火、烟雾、积水 | 88%+ |
| 设备检测 | 检测灭火器、警示标志 | 90%+ |
| 人员检测 | 检测人员数量和位置 | 95%+ |
| 伪造检测 | 检测截图、翻拍、PS痕迹 | 85%+ |

```mermaid
flowchart TD
    Photo[上传照片] --> AI[AI审查引擎]

    AI --> PPE{PPE检测}
    AI --> Env{环境检测}
    AI --> Fake{伪造检测}

    PPE -->|通过| Pass1[✅]
    PPE -->|未通过| Fail1[❌ 未佩戴安全帽]

    Env -->|通过| Pass2[✅]
    Env -->|未通过| Fail2[❌ 检测到明火风险]

    Fake -->|通过| Pass3[✅]
    Fake -->|未通过| Fail3[❌ 疑似非实时拍摄]

    Pass1 & Pass2 & Pass3 --> Result{综合结果}
    Fail1 & Fail2 & Fail3 --> Result

    Result -->|全部通过| Accept[接受照片]
    Result -->|存在问题| Review[人工复核]
```

## 10.3 多级覆盖机制

### 10.3.1 覆盖层级模型

```mermaid
flowchart TD
    subgraph "Level 1: 系统默认"
        L1[系统标准模板<br/>所有企业共享]
    end

    subgraph "Level 2: 公司定制"
        L2[公司级配置<br/>覆盖系统默认]
    end

    subgraph "Level 3: 项目定制"
        L3[项目级配置<br/>覆盖公司配置]
    end

    L1 -->|继承| L2
    L2 -->|继承| L3
    L3 --> Final[最终生效配置]

    style L1 fill:#e1f5ff
    style L2 fill:#fff4e6
    style L3 fill:#e8f5e9
```

### 10.3.2 覆盖规则

| 配置项 | Level 1 系统 | Level 2 公司 | Level 3 项目 | 最终生效 |
|-------|:----------:|:----------:|:----------:|:-------:|
| 字段定义 | 基础字段集 | 可新增字段 | 可新增字段 | 合并所有 |
| 字段属性 | 默认属性 | 可修改 | 可修改 | 最低级优先 |
| 布局配置 | 标准布局 | 可替换 | 可替换 | 最低级优先 |
| 校验规则 | 基础规则 | 可追加 | 可追加 | 合并所有 |
| 硬约束 | 系统强制 | 不可修改 | 不可修改 | 系统级 |
| 状态机 | 标准流程 | 不可修改 | 不可修改 | 系统级 |

### 10.3.3 覆盖配置示例

```json
{
  "override": {
    "level": 2,
    "org_id": "ORG-001",
    "base_metadata": "hot_work_standard_v2",
    "changes": {
      "schema": {
        "add_fields": [
          {
            "key": "company_specific_check",
            "type": "checkbox",
            "label": "公司特殊安全检查项",
            "required": true,
            "after": "safety_measures"
          }
        ],
        "modify_fields": [
          {
            "key": "gas_oxygen",
            "changes": {
              "min": 19.5,
              "max": 23.0,
              "errorMessage": "本公司要求氧气浓度在19.5%-23.0%之间（严于国标）"
            }
          }
        ]
      },
      "layout": null,
      "rules": {
        "add_rules": [
          {
            "type": "validation",
            "field": "work_time_end",
            "expr": "DATE_DIFF(data.work_time_end, data.work_time_start, 'hours') <= 8",
            "message": "本公司规定单次作业时长不超过8小时"
          }
        ]
      }
    }
  }
}
```

### 10.3.4 覆盖冲突处理

```mermaid
flowchart TD
    Conflict[检测到覆盖冲突] --> Type{冲突类型}

    Type -->|同字段不同属性| Merge[属性级合并<br/>低级覆盖高级]
    Type -->|同规则不同条件| Append[规则追加<br/>两条规则都生效]
    Type -->|硬约束被覆盖| Block[阻止覆盖<br/>硬约束不可修改]
    Type -->|布局冲突| Replace[整体替换<br/>低级布局生效]

    Block --> Alert[提示管理员<br/>硬约束不可修改]
```
